# 1.1 Docker 核心概念

> 本节目标：理解 Docker 核心概念，掌握 Dockerfile 编写和 multi-stage build，能用 docker compose 编排多服务环境。

---

## 一、核心概念

### 1. 镜像（Image）

镜像是一个**只读模板**，包含运行应用所需的一切：代码、运行时、库、环境变量、配置文件。

```bash
# 查看本地镜像
docker images

# 拉取镜像
docker pull nginx:1.25

# 删除镜像
docker rmi nginx:1.25
```

### 2. 容器（Container）

容器是镜像的**运行实例**，拥有自己的文件系统、网络和进程空间。

```bash
# 运行容器
docker run -d --name my-nginx -p 8080:80 nginx:1.25

# 查看运行中的容器
docker ps

# 查看所有容器（含已停止）
docker ps -a

# 进入容器
docker exec -it my-nginx /bin/sh

# 停止/删除容器
docker stop my-nginx
docker rm my-nginx
```

### 3. 镜像分层原理

Docker 镜像由多个**只读层**叠加而成，每条 Dockerfile 指令创建一层：

```
┌─────────────────────────┐
│  Layer 4: CMD (可写层)    │  ← 容器层（运行时）
├─────────────────────────┤
│  Layer 3: COPY app      │  ← 应用代码
├─────────────────────────┤
│  Layer 2: RUN apt-get   │  ← 依赖安装
├─────────────────────────┤
│  Layer 1: FROM ubuntu   │  ← 基础镜像
└─────────────────────────┘
```

**关键特性**：
- 每层只读，容器运行时在顶部添加可写层（Copy-on-Write）
- 多个容器共享相同的底层镜像层，节省磁盘空间
- 构建缓存：如果某层未改变，Docker 会复用缓存

### 4. Union FS（联合文件系统）

Union FS 是分层的底层实现，将多个目录"联合"挂载到同一个目录：

- **OverlayFS**（Linux 默认）：分为 lowerdir（只读层）和 upperdir（可写层），通过 merged 目录呈现统一视图
- 写时复制（CoW）：修改只读层文件时，先复制到可写层再修改

```bash
# 查看 Docker 使用的存储驱动
docker info | grep "Storage Driver"
# 输出: Storage Driver: overlay2
```

## 二、Dockerfile 指令详解

### 完整指令参考

| 指令 | 作用 | 示例 |
|------|------|------|
| `FROM` | 指定基础镜像 | `FROM golang:1.22-alpine` |
| `WORKDIR` | 设置工作目录 | `WORKDIR /app` |
| `COPY` | 复制文件到镜像 | `COPY . .` |
| `ADD` | 复制文件（支持 URL 和解压） | `ADD archive.tar.gz /app/` |
| `RUN` | 执行命令（构建时） | `RUN go build -o server` |
| `CMD` | 容器启动默认命令 | `CMD ["./server"]` |
| `ENTRYPOINT` | 容器启动入口点 | `ENTRYPOINT ["./server"]` |
| `ENV` | 设置环境变量 | `ENV GO_ENV=production` |
| `ARG` | 构建时变量 | `ARG VERSION=1.0` |
| `EXPOSE` | 声明端口（仅文档作用） | `EXPOSE 8080` |
| `VOLUME` | 声明挂载点 | `VOLUME /data` |
| `USER` | 指定运行用户 | `USER nonroot` |

### COPY vs ADD 区别

| 特性 | COPY | ADD |
|------|------|-----|
| 复制本地文件 | ✅ | ✅ |
| 自动解压 tar | ❌ | ✅ |
| 支持 URL 下载 | ❌ | ✅（不推荐） |
| **推荐程度** | **优先使用** | 仅需解压时使用 |

```dockerfile
# 推荐：用 COPY
COPY server.yaml /app/config/

# 仅在需要自动解压时用 ADD
ADD game-assets.tar.gz /app/assets/
```

### CMD vs ENTRYPOINT 区别

| 特性 | CMD | ENTRYPOINT |
|------|-----|------------|
| 可被 `docker run` 参数覆盖 | ✅ 整体替换 | ❌ 参数追加到后面 |
| 适用场景 | 提供默认命令，允许覆盖 | 固定入口，参数可变 |

```dockerfile
# 方式 1：CMD（可覆盖）
CMD ["./server", "--port=8080"]
# docker run myimage                → ./server --port=8080
# docker run myimage --port=9090    → --port=9090（CMD 被替换）

# 方式 2：ENTRYPOINT + CMD 组合（推荐）
ENTRYPOINT ["./server"]
CMD ["--port=8080"]
# docker run myimage                → ./server --port=8080
# docker run myimage --port=9090    → ./server --port=9090（CMD 部分被替换）
```

### .dockerignore

类似 `.gitignore`，排除不需要发送到构建上下文的文件：

```
# .dockerignore
.git
.gitignore
README.md
docker-compose*.yml
*.test
vendor/
node_modules/
.env
```

**作用**：减少构建上下文大小 → 加快构建速度，避免敏感文件泄露到镜像。

## 三、Multi-stage Build（多阶段构建）

### 原理

使用多个 `FROM` 指令，在前面的阶段编译，只将最终产物复制到最小的运行镜像中。

### 对比

```dockerfile
# ❌ 单阶段构建 —— 镜像包含编译工具链，体积大（~800MB）
FROM golang:1.22
WORKDIR /app
COPY . .
RUN go build -o server
CMD ["./server"]
```

```dockerfile
# ✅ 多阶段构建 —— 最终镜像仅包含二进制，体积小（~15MB）
# 阶段 1：编译
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -ldflags="-s -w" -o server

# 阶段 2：运行
FROM alpine:3.19
RUN apk --no-cache add ca-certificates
WORKDIR /app
COPY --from=builder /app/server .
COPY --from=builder /app/config ./config
EXPOSE 8080
CMD ["./server"]
```

## 四、实操练习

### 练习 1：为 Go 游戏服务编写 Dockerfile

以一个简单的游戏网关服务为例：

**目录结构**：
```
game-gate/
├── main.go
├── go.mod
├── go.sum
├── config/
│   └── server.yaml
├── Dockerfile
└── .dockerignore
```

**main.go**（示例）：
```go
package main

import (
    "fmt"
    "log"
    "net/http"
    "os"
)

func main() {
    port := os.Getenv("PORT")
    if port == "" {
        port = "8080"
    }

    http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
        fmt.Fprintf(w, "OK")
    })

    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintf(w, "Game Gate Server v1.0")
    })

    log.Printf("Game Gate starting on port %s", port)
    log.Fatal(http.ListenAndServe(":"+port, nil))
}
```

**go.mod**：
```
module game-gate

go 1.22
```

**config/server.yaml**：
```yaml
server:
  name: game-gate
  port: 8080
  max_connections: 10000
redis:
  addr: redis:6379
mysql:
  dsn: root:password@tcp(mysql:3306)/gamedb
```

**Dockerfile**：
```dockerfile
# 阶段 1：编译
FROM golang:1.22-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o game-gate

# 阶段 2：运行
FROM alpine:3.19
RUN apk --no-cache add ca-certificates tzdata \
    && cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
WORKDIR /app
COPY --from=builder /app/game-gate .
COPY --from=builder /app/config ./config
EXPOSE 8080
CMD ["./game-gate"]
```

**.dockerignore**：
```
.git
README.md
Dockerfile
docker-compose.yml
*.test
```

**构建与验证**：
```bash
# 构建
docker build -t game-gate:v1.0 .

# 查看镜像大小（目标 < 100MB）
docker images game-gate
# REPOSITORY   TAG    IMAGE ID       CREATED         SIZE
# game-gate    v1.0   abc123...      10 seconds ago  25.3MB  ✅

# 运行测试
docker run -d --name gate -p 8080:8080 game-gate:v1.0

# 验证
curl http://localhost:8080/health
# OK

curl http://localhost:8080/
# Game Gate Server v1.0

# 清理
docker stop gate && docker rm gate
```

### 练习 2：Docker Compose 编排多服务环境

**docker-compose.yml**：
```yaml
version: "3.9"

services:
  game-gate:
    build: ./game-gate
    image: game-gate:v1.0
    ports:
      - "8080:8080"
    environment:
      - PORT=8080
    depends_on:
      redis:
        condition: service_healthy
      mysql:
        condition: service_healthy
    networks:
      - game-net
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - game-net
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: gamedb
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - game-net
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      timeout: 3s
      retries: 10

volumes:
  redis-data:
  mysql-data:

networks:
  game-net:
    driver: bridge
```

**启动与验证**：
```bash
# 一键启动
docker compose up -d

# 查看状态
docker compose ps
# NAME         SERVICE      STATUS    PORTS
# game-gate-1  game-gate    running   0.0.0.0:8080->8080/tcp
# redis-1      redis        running   0.0.0.0:6379->6379/tcp
# mysql-1      mysql        running   0.0.0.0:3306->3306/tcp

# 验证服务间通信
docker compose exec game-gate sh -c "wget -qO- http://localhost:8080/health"
# OK

# 查看日志
docker compose logs -f game-gate

# 停止
docker compose down

# 停止并删除 volume（慎用）
docker compose down -v
```
