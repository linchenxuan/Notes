# 2.2 核心资源对象 — Pod / Deployment / Service

> 本节目标：掌握 K8s 最核心的三个资源对象，能独立完成部署、更新和回滚。

---

## 一、Pod

### 概念

Pod 是 K8s 的**最小调度单元**，包含一个或多个共享网络和存储的容器。

### Pod 生命周期

```
Pending → Running → Succeeded / Failed
                 ↘ Unknown（节点失联）
```

| 状态 | 含义 |
|------|------|
| Pending | 已被调度系统接受，镜像正在拉取或等待调度 |
| Running | 至少一个容器在运行 |
| Succeeded | 所有容器正常退出（exit 0），不会重启 |
| Failed | 至少一个容器非正常退出 |
| Unknown | 无法获取状态（通常是节点通信问题） |

### Pod YAML 结构

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: game-gate
  labels:
    app: game-gate
    tier: frontend
spec:
  containers:
  - name: gate
    image: game-gate:v1.0
    ports:
    - containerPort: 8080
    env:
    - name: PORT
      value: "8080"
    resources:
      requests:
        cpu: "100m"
        memory: "128Mi"
      limits:
        cpu: "500m"
        memory: "256Mi"
    livenessProbe:         # 存活探针：失败则重启容器
      httpGet:
        path: /health
        port: 8080
      initialDelaySeconds: 5
      periodSeconds: 10
    readinessProbe:        # 就绪探针：失败则从 Service 移除
      httpGet:
        path: /ready
        port: 8080
      initialDelaySeconds: 3
      periodSeconds: 5
  restartPolicy: Always    # Always | OnFailure | Never
```

## 二、Deployment

### 概念

Deployment 管理 ReplicaSet，ReplicaSet 管理 Pod。提供声明式更新、滚动发布和回滚能力。

```
Deployment → ReplicaSet → Pod, Pod, Pod
```

### 滚动更新策略

```yaml
spec:
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1          # 更新时最多多出 1 个 Pod
      maxUnavailable: 0    # 更新时不允许有 Pod 不可用
```

| 参数 | 含义 | 推荐值 |
|------|------|--------|
| maxSurge | 超出 replicas 的最大 Pod 数 | 1 或 25% |
| maxUnavailable | 更新中不可用的最大 Pod 数 | 0（零停机） |

### 实操：部署游戏网关服务

**game-gate-deployment.yaml**：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game-gate
  labels:
    app: game-gate
spec:
  replicas: 3
  selector:
    matchLabels:
      app: game-gate
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: game-gate
        version: v1
    spec:
      containers:
      - name: gate
        image: nginx:1.25    # 用 nginx 代替实际游戏服务演示
        ports:
        - containerPort: 80
        resources:
          requests:
            cpu: "100m"
            memory: "64Mi"
          limits:
            cpu: "200m"
            memory: "128Mi"
        livenessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 3
          periodSeconds: 5
```

```bash
# 部署
kubectl apply -f game-gate-deployment.yaml

# 查看 Deployment
kubectl get deployment game-gate
# NAME        READY   UP-TO-DATE   AVAILABLE   AGE
# game-gate   3/3     3            3           30s

# 查看 Pod
kubectl get pods -l app=game-gate
# NAME                         READY   STATUS    RESTARTS   AGE
# game-gate-6d8f9b7c5d-abc12   1/1     Running   0          30s
# game-gate-6d8f9b7c5d-def34   1/1     Running   0          30s
# game-gate-6d8f9b7c5d-ghi56   1/1     Running   0          30s

# 查看 ReplicaSet
kubectl get rs -l app=game-gate

# 查看详情
kubectl describe deployment game-gate
```

## 三、Service

### 概念

Service 为一组 Pod 提供稳定的访问入口（固定 IP + DNS 名）和负载均衡。

### Service 类型

| 类型           | 访问范围          | 使用场景        |
| ------------ | ------------- | ----------- |
| ClusterIP    | 集群内部          | 内部服务间通信（默认） |
| NodePort     | 集群外部（节点IP:端口） | 开发测试        |
| LoadBalancer | 集群外部（云负载均衡器）  | 生产环境暴露服务    |
| ExternalName | DNS CNAME 映射  | 引用外部服务      |

### selector 机制

Service 通过 label selector 选中 Pod，自动维护 Endpoints：

```
Service (selector: app=game-gate)
    │
    ├── Endpoints: 10.244.0.5:80
    ├── Endpoints: 10.244.0.6:80
    └── Endpoints: 10.244.0.7:80
```

### 实操：创建 Service

**game-gate-service.yaml**：

```yaml
apiVersion: v1
kind: Service
metadata:
  name: game-gate-svc
spec:
  type: ClusterIP
  selector:
    app: game-gate
  ports:
  - port: 80          # Service 端口
    targetPort: 80     # Pod 端口
    protocol: TCP
```

```bash
# 创建 Service
kubectl apply -f game-gate-service.yaml

# 查看 Service
kubectl get svc game-gate-svc
# NAME            TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)   AGE
# game-gate-svc   ClusterIP   10.96.123.45    <none>        80/TCP    5s

# 查看 Endpoints
kubectl get endpoints game-gate-svc
# NAME            ENDPOINTS                                   AGE
# game-gate-svc   10.244.0.5:80,10.244.0.6:80,10.244.0.7:80  5s

# 本地访问（port-forward）
kubectl port-forward svc/game-gate-svc 8080:80
# 浏览器访问 http://localhost:8080

# NodePort 方式
# 修改 type: NodePort，会自动分配 30000-32767 范围的端口
```

## 四、实操：滚动更新与回滚

### 滚动更新

```bash
# 方式 1：修改 YAML 中的镜像版本后 apply
# 将 image: nginx:1.25 改为 image: nginx:1.26
kubectl apply -f game-gate-deployment.yaml

# 方式 2：命令行直接修改
kubectl set image deployment/game-gate gate=nginx:1.26

# 观察滚动更新过程
kubectl rollout status deployment/game-gate
# Waiting for deployment "game-gate" rollout to finish: 1 out of 3 new replicas have been updated...
# Waiting for deployment "game-gate" rollout to finish: 2 out of 3 new replicas have been updated...
# deployment "game-gate" successfully rolled out

# 查看更新过程中的 Pod 变化
kubectl get pods -l app=game-gate -w
# 会看到新 Pod 逐步创建，旧 Pod 逐步终止
```

### 回滚

```bash
# 查看发布历史
kubectl rollout history deployment/game-gate
# REVISION  CHANGE-CAUSE
# 1         <none>
# 2         <none>

# 回滚到上一版本
kubectl rollout undo deployment/game-gate

# 回滚到指定版本
kubectl rollout undo deployment/game-gate --to-revision=1

# 验证镜像版本已恢复
kubectl describe deployment game-gate | grep Image
# Image: nginx:1.25
```

### 验证滚动更新不中断（脚本）

```bash
# 终端 1：持续发送请求
while true; do
  curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8080
  sleep 0.5
done
# 应持续输出 200，无 5xx 错误

# 终端 2：执行更新
kubectl set image deployment/game-gate gate=nginx:1.26
```

## 五、常用 kubectl 命令速查

```bash
# 资源操作
kubectl get pods/svc/deploy/rs/nodes     # 查看资源
kubectl describe pod <name>               # 详细信息
kubectl logs <pod-name> [-f]              # 查看日志
kubectl exec -it <pod-name> -- /bin/sh    # 进入容器
kubectl delete pod <name>                 # 删除 Pod

# Deployment 操作
kubectl scale deploy/<name> --replicas=5  # 手动扩缩容
kubectl rollout status deploy/<name>      # 查看发布状态
kubectl rollout history deploy/<name>     # 查看历史
kubectl rollout undo deploy/<name>        # 回滚

# 调试
kubectl get events --sort-by='.lastTimestamp'  # 查看事件
kubectl top pod                                # 资源使用（需 metrics-server）
kubectl get pods -o wide                       # 查看 Pod 所在节点和 IP
```
