# 2.3 配置与存储 — ConfigMap / Secret / PV / PVC

> 本节目标：掌握 K8s 配置注入和持久化存储机制。

---

## 一、ConfigMap

### 概念

ConfigMap 用于将**非敏感配置数据**与 Pod 解耦，支持以环境变量或文件挂载的方式注入。

### 创建方式

```bash
# 方式 1：从键值对创建
kubectl create configmap game-config \
  --from-literal=server.name=game-gate \
  --from-literal=server.port=8080

# 方式 2：从文件创建
kubectl create configmap game-config --from-file=server.yaml

# 方式 3：YAML 声明式
```

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: game-config
data:
  server.name: "game-gate"
  server.port: "8080"
  server.yaml: |
    server:
      name: game-gate
      port: 8080
      max_connections: 10000
    redis:
      addr: redis:6379
    match:
      timeout: 30s
```

### 注入方式

#### 方式 1：环境变量

```yaml
spec:
  containers:
  - name: gate
    image: game-gate:v1.0
    env:
    - name: SERVER_NAME
      valueFrom:
        configMapKeyRef:
          name: game-config
          key: server.name
    # 或一次注入所有键
    envFrom:
    - configMapRef:
        name: game-config
```

#### 方式 2：Volume 挂载（推荐，支持热更新）

```yaml
spec:
  containers:
  - name: gate
    image: game-gate:v1.0
    volumeMounts:
    - name: config-volume
      mountPath: /app/config
      readOnly: true
  volumes:
  - name: config-volume
    configMap:
      name: game-config
      items:                    # 可选：只挂载指定 key
      - key: server.yaml
        path: server.yaml       # 容器内文件名
```

### 热更新机制

- **Volume 挂载**：ConfigMap 更新后，kubelet 会自动更新 Pod 内的文件（延迟约 1-2 分钟）
- **环境变量**：不会自动更新，需重启 Pod

```bash
# 更新 ConfigMap
kubectl edit configmap game-config
# 或
kubectl apply -f updated-configmap.yaml

# Volume 挂载方式：等待约 1-2 分钟后验证
kubectl exec <pod-name> -- cat /app/config/server.yaml
```

## 二、Secret

### 概念

Secret 用于存储**敏感数据**（密码、Token、证书等），数据以 base64 编码存储。

> 注意：base64 只是编码，不是加密。生产环境应结合 RBAC 和加密存储。

### Secret 类型

| 类型 | 用途 |
|------|------|
| Opaque | 通用（默认） |
| kubernetes.io/dockerconfigjson | 镜像仓库认证 |
| kubernetes.io/tls | TLS 证书 |
| kubernetes.io/basic-auth | Basic 认证 |

### 创建方式

```bash
# 命令行创建
kubectl create secret generic db-secret \
  --from-literal=username=root \
  --from-literal=password=GamePass123

# 查看（base64 编码）
kubectl get secret db-secret -o yaml
# data:
#   password: R2FtZVBhc3MxMjM=    ← base64("GamePass123")
#   username: cm9vdA==             ← base64("root")
```

```yaml
# YAML 声明式（值需要手动 base64 编码）
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
type: Opaque
data:
  username: cm9vdA==           # echo -n "root" | base64
  password: R2FtZVBhc3MxMjM=  # echo -n "GamePass123" | base64
---
# 也可以用 stringData 自动编码
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
type: Opaque
stringData:
  username: root
  password: GamePass123
```

### 注入方式

```yaml
spec:
  containers:
  - name: game-server
    image: game-server:v1.0
    env:
    - name: DB_USERNAME
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: username
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: password
```

## 三、PV / PVC / StorageClass

### 概念关系

```
StorageClass（定义存储类型：SSD/HDD、供应商）
      │
      │ 动态供应
      ▼
PersistentVolume（PV）—— 实际存储资源（管理员/动态创建）
      ▲
      │ 绑定
      │
PersistentVolumeClaim（PVC）—— 用户的存储请求（Pod 使用）
```

### 访问模式

| 模式 | 缩写 | 含义 |
|------|------|------|
| ReadWriteOnce | RWO | 单节点读写 |
| ReadOnlyMany | ROX | 多节点只读 |
| ReadWriteMany | RWX | 多节点读写 |

### 回收策略

| 策略 | 含义 |
|------|------|
| Retain | PVC 删除后 PV 保留数据，需手动清理 |
| Delete | PVC 删除后 PV 和底层存储一起删除 |
| Recycle | 已废弃 |

## 四、完整实操

### 实操 1：ConfigMap 挂载游戏配置

```yaml
# game-configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: game-server-config
data:
  server.yaml: |
    server:
      name: game-gate
      port: 8080
      max_connections: 10000
    redis:
      addr: redis-svc:6379
    match:
      timeout: 30s
      max_players: 10
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game-gate
spec:
  replicas: 2
  selector:
    matchLabels:
      app: game-gate
  template:
    metadata:
      labels:
        app: game-gate
    spec:
      containers:
      - name: gate
        image: nginx:1.25
        volumeMounts:
        - name: config
          mountPath: /app/config
          readOnly: true
      volumes:
      - name: config
        configMap:
          name: game-server-config
```

```bash
kubectl apply -f game-configmap.yaml

# 验证配置已挂载
kubectl exec deploy/game-gate -- cat /app/config/server.yaml

# 修改 ConfigMap
kubectl edit configmap game-server-config
# 修改 max_connections 为 20000

# 等待约 1-2 分钟后验证更新
kubectl exec deploy/game-gate -- cat /app/config/server.yaml
# 应看到 max_connections: 20000 ✅
```

### 实操 2：Secret 注入数据库密码

```yaml
# db-secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: mysql-secret
type: Opaque
stringData:
  root-password: GamePass123
  database: gamedb
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game-server
spec:
  replicas: 1
  selector:
    matchLabels:
      app: game-server
  template:
    metadata:
      labels:
        app: game-server
    spec:
      containers:
      - name: server
        image: nginx:1.25
        env:
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: root-password
        - name: DB_NAME
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: database
```

```bash
kubectl apply -f db-secret.yaml

# 验证 Secret（base64 编码）
kubectl get secret mysql-secret -o yaml
# data:
#   root-password: R2FtZVBhc3MxMjM=

# 验证 Pod 内能读取明文
kubectl exec deploy/game-server -- env | grep DB_
# DB_PASSWORD=GamePass123  ✅
# DB_NAME=gamedb           ✅
```

### 实操 3：PVC 给 MySQL StatefulSet 使用

```yaml
# mysql-statefulset.yaml
apiVersion: v1
kind: Service
metadata:
  name: mysql-svc
spec:
  clusterIP: None    # Headless Service
  selector:
    app: mysql
  ports:
  - port: 3306
---
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mysql
spec:
  serviceName: mysql-svc
  replicas: 1
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - name: mysql
        image: mysql:8.0
        ports:
        - containerPort: 3306
        env:
        - name: MYSQL_ROOT_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: root-password
        - name: MYSQL_DATABASE
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: database
        volumeMounts:
        - name: mysql-data
          mountPath: /var/lib/mysql
  volumeClaimTemplates:
  - metadata:
      name: mysql-data
    spec:
      accessModes: ["ReadWriteOnce"]
      resources:
        requests:
          storage: 5Gi
```

```bash
kubectl apply -f mysql-statefulset.yaml

# 查看 PVC
kubectl get pvc
# NAME                 STATUS   VOLUME         CAPACITY   ACCESS MODES
# mysql-data-mysql-0   Bound    pvc-xxx        5Gi        RWO

# 写入数据
kubectl exec mysql-0 -- mysql -pGamePass123 gamedb \
  -e "CREATE TABLE test(id INT); INSERT INTO test VALUES(1);"

# 删除 Pod（StatefulSet 自动重建）
kubectl delete pod mysql-0

# 等待重建完成
kubectl wait --for=condition=ready pod/mysql-0 --timeout=120s

# 验证数据保留
kubectl exec mysql-0 -- mysql -pGamePass123 gamedb -e "SELECT * FROM test;"
# +------+
# | id   |
# +------+
# |    1 |
# +------+
# ✅ 数据完整保留
```
