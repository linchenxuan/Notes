## 二、控制面组件详解

### 1. API Server（kube-apiserver）

**职责**：K8s 的"前门"，所有操作的唯一入口。

- 接收并验证所有 REST 请求（kubectl、SDK、其他组件）
- 认证（Authentication）→ 授权（Authorization）→ 准入控制（Admission Control）
- 是唯一直接与 etcd 通信的组件
- 提供 Watch 机制，通知其他组件资源变化

### 2. etcd

**职责**：分布式键值存储，保存集群所有状态数据。

- 存储所有 K8s 资源对象（Pod、Service、ConfigMap 等）
- 使用 Raft 协议保证一致性
- 生产环境建议 3 或 5 节点高可用部署

### 3. Scheduler（kube-scheduler）

**职责**：决定 Pod 在哪个 Node 上运行。

调度流程：
1. **过滤（Filtering）**：排除不满足条件的 Node（资源不足、Taint 不匹配等）
2. **打分（Scoring）**：对候选 Node 评分（资源均衡、亲和性等）
3. **绑定（Binding）**：将 Pod 绑定到得分最高的 Node

### 4. Controller Manager（kube-controller-manager）

**职责**：运行各种控制器，确保集群实际状态趋近期望状态。

核心控制器：
- **ReplicaSet Controller**：维护 Pod 副本数
- **Deployment Controller**：管理 Deployment 的滚动更新
- **Node Controller**：监控节点健康状态
- **Job Controller**：管理一次性任务
- **ServiceAccount Controller**：为新命名空间创建默认 SA

**控制循环（Reconciliation Loop）**：

```
  观察当前状态 → 对比期望状态 → 执行修正操作 → 重复
```

## 三、数据面组件详解

### 1. kubelet

**职责**：每个 Node 上的"代理人"，负责管理 Pod 生命周期。

- 接收 API Server 下发的 PodSpec
- 通过 CRI（Container Runtime Interface）调用容器运行时
- 汇报 Node 状态和 Pod 状态
- 执行健康检查（liveness / readiness / startup probe）

### 2. kube-proxy

**职责**：维护 Node 上的网络规则，实现 Service 的负载均衡。

工作模式：
- **iptables 模式**（默认）：通过 iptables 规则转发流量
- **IPVS 模式**：大规模集群推荐，性能更好
- **nftables 模式**：新一代，逐步替代 iptables

## 四、创建 Pod 时的组件协作流程

```
1. kubectl apply → API Server（验证 + 存 etcd）
2. Scheduler watch 到未调度 Pod → 选择 Node → 更新 Pod.spec.nodeName
3. 目标 Node 的 kubelet watch 到分配给自己的 Pod
4. kubelet → CRI → containerd → 拉取镜像 → 创建容器
5. kubelet 汇报 Pod 状态 → API Server → etcd
6. kube-proxy 更新 iptables 规则（如 Pod 属于某个 Service）
```

## 五、本地集群搭建

### 方式 1：minikube

```bash
# 安装 minikube（Linux）
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# 启动集群
minikube start --driver=docker --cpus=2 --memory=4096

# 验证
minikube status
# minikube
# type: Control Plane
# host: Running
# kubelet: Running
# apiserver: Running
# kubeconfig: Configured
```

### 方式 2：kind（Kubernetes in Docker）

```bash
# 安装 kind
# Linux
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.22.0/kind-linux-amd64
chmod +x ./kind
sudo mv ./kind /usr/local/bin/kind

# 创建集群
kind create cluster --name k8s-lab

# 多节点集群配置
cat <<EOF | kind create cluster --name k8s-lab --config=-
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
- role: control-plane
- role: worker
- role: worker
EOF
```

### 安装 kubectl

```bash
# Linux
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
sudo mv kubectl /usr/local/bin/

# 配置自动补全（强烈推荐）
echo 'source <(kubectl completion bash)' >> ~/.bashrc
echo 'alias k=kubectl' >> ~/.bashrc
echo 'complete -o default -F __start_kubectl k' >> ~/.bashrc
source ~/.bashrc
```
