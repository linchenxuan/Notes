## Week 10：Service类型和负载均衡

### 🌐 Day 1-2：Service基础概念回顾

#### Service四种类型详解
```yaml
# 我的Service学习笔记

# 1. ClusterIP（默认，集群内部访问）
apiVersion: v1
kind: Service
metadata:
  name: clusterip-service
spec:
  type: ClusterIP  # 默认类型
  clusterIP: None  # Headless Service
  ports:
  - port: 80
    targetPort: 8080
  selector:
    app: webapp

---
# 2. NodePort（节点端口暴露）
apiVersion: v1
kind: Service
metadata:
  name: nodeport-service
spec:
  type: NodePort
  ports:
  - port: 80      # Service端口
    targetPort: 8080  # Pod端口
    nodePort: 30080   # 节点端口（30000-32767）
  selector:
    app: webapp

---
# 3. LoadBalancer（云负载均衡器）
apiVersion: v1
kind: Service
metadata:
  name: loadbalancer-service
  annotations:
    service.beta.kubernetes.io/aws-load-balancer-type: "nlb"
spec:
  type: LoadBalancer
  ports:
  - port: 80
    targetPort: 8080
  selector:
    app: webapp

---
# 4. ExternalName（外部服务别名）
apiVersion: v1
kind: Service
metadata:
  name: externalname-service
spec:
  type: ExternalName
  externalName: api.external-service.com
```

#### Service工作原理理解
```bash
# 我的Service实践记录

# 1. 创建测试Deployment
kubectl create deployment webapp --image=nginx:1.23 --replicas=3

# 2. 创建不同类型的Service
kubectl apply -f clusterip-service.yaml
kubectl apply -f nodeport-service.yaml

# 3. 查看Service详情
kubectl get services
kubectl describe service clusterip-service

# 4. 测试ClusterIP访问
kubectl run test-pod --image=busybox --rm -it -- wget -qO- http://clusterip-service

# 5. 测试NodePort访问
minikube service nodeport-service --url
# 获取访问URL：http://192.168.49.2:30080

# 6. 查看Endpoints
kubectl get endpoints clusterip-service
# 可以看到后端Pod的IP列表

# 7. 查看Service的iptables规则（在节点上）
# sudo iptables -t nat -L KUBE-SERVICES
```

**我的理解总结**：
- **ClusterIP**：集群内部虚拟IP，只能在集群内访问
- **NodePort**：在每个节点上开放指定端口，外部可以通过节点IP访问
- **LoadBalancer**：在云环境中创建外部负载均衡器
- **ExternalName**：将服务映射到外部DNS名称

### ⚖️ Day 3-4：负载均衡算法和会话保持

#### Service负载均衡配置
```yaml
# 我的负载均衡学习笔记
apiVersion: v1
kind: Service
metadata:
  name: webapp-service
spec:
  sessionAffinity: ClientIP  # 会话保持策略
  sessionAffinityConfig:
    clientIP:
      timeoutSeconds: 10800  # 会话保持时间（3小时）
  
  # 负载均衡算法（kube-proxy模式相关）
  # iptables模式：随机选择
  # IPVS模式：支持多种算法
  
  ports:
  - port: 80
    targetPort: 8080
    protocol: TCP
  selector:
    app: webapp
```

#### IPVS模式配置（高级）
```yaml
# 我的IPVS配置学习
# 需要在kube-proxy中配置
apiVersion: v1
kind: ConfigMap
metadata:
  name: kube-proxy
  namespace: kube-system
data:
  config.conf: |
    apiVersion: kubeproxy.config.k8s.io/v1alpha1
    kind: KubeProxyConfiguration
    mode: "ipvs"  # 使用IPVS模式
    ipvs:
      scheduler: "rr"  # 轮询算法
      # 其他算法：lc（最少连接）、dh（目的地址哈希）、sh（源地址哈希）
      # sed（最短预期延迟）、nq（永不排队）
```

#### 会话保持测试

## Week 11：Ingress控制器配置

### 🚪 Day 7-8：Ingress基础概念

#### Ingress资源配置
```yaml
# 我的Ingress学习笔记
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: webapp-ingress
  annotations:
    # Nginx Ingress控制器注解
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    nginx.ingress.kubernetes.io/use-regex: "true"
spec:
  # 1. Ingress类
  ingressClassName: nginx
  
  # 2. TLS配置
  tls:
  - hosts:
    - app.example.com
    - api.example.com
    secretName: tls-secret
  
  # 3. 规则配置
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: webapp-service
            port:
              number: 80
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 8080
  
  - host: api.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 8080
```

#### 安装Nginx Ingress控制器

### ⚙️ Day 9-10：高级Ingress配置

#### 路径重写和负载均衡
```yaml
# 我的高级Ingress配置
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: advanced-ingress
  annotations:
    # 路径重写
    nginx.ingress.kubernetes.io/rewrite-target: /$2
    
    # 负载均衡算法
    nginx.ingress.kubernetes.io/upstream-hash-by: "$remote_addr"
    
    # 连接限制
    nginx.ingress.kubernetes.io/limit-connections: "10"
    nginx.ingress.kubernetes.io/limit-rps: "5"
    
    # 超时设置
    nginx.ingress.kubernetes.io/proxy-connect-timeout: "30"
    nginx.ingress.kubernetes.io/proxy-send-timeout: "60"
    nginx.ingress.kubernetes.io/proxy-read-timeout: "60"
    
    # 缓存配置
    nginx.ingress.kubernetes.io/proxy-buffering: "on"
    nginx.ingress.kubernetes.io/proxy-buffer-size: "8k"
    
    # 认证配置
    nginx.ingress.kubernetes.io/auth-type: basic
    nginx.ingress.kubernetes.io/auth-secret: basic-auth
    nginx.ingress.kubernetes.io/auth-realm: 'Authentication Required'
    
    # CORS配置
    nginx.ingress.kubernetes.io/enable-cors: "true"
    nginx.ingress.kubernetes.io/cors-allow-origin: "*"
    nginx.ingress.kubernetes.io/cors-allow-methods: "GET, POST, PUT, DELETE, OPTIONS"
    
spec:
  ingressClassName: nginx
  rules:
  - host: advanced.example.com
    http:
      paths:
      - path: /app(/|$)(.*)
        pathType: Prefix
        backend:
          service:
            name: webapp-service
            port:
              number: 80
      - path: /api(/|$)(.*)
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 8080
```

## Week 12：网络策略NetworkPolicy

### 🔒 Day 11-12：网络策略基础

#### NetworkPolicy基础配置
```yaml
# 我的NetworkPolicy学习笔记
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: webapp-network-policy
spec:
  # 1. Pod选择器（应用此策略的Pod）
  podSelector:
    matchLabels:
      app: webapp
  
  # 2. 策略类型
  policyTypes:
  - Ingress  # 入站规则
  - Egress   # 出站规则
  
  # 3. 入站规则
  ingress:
  - from:
    - podSelector:  # 来自哪些Pod
        matchLabels:
          app: frontend
    - namespaceSelector:  # 来自哪些命名空间
        matchLabels:
          env: production
    - ipBlock:  # 来自哪些IP段
        cidr: 10.0.0.0/8
        except:  # 排除的IP段
        - 10.0.1.0/24
    ports:
    - protocol: TCP
      port: 8080
  
  # 4. 出站规则
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 5432
  - to:
    - ipBlock:  # 允许访问外部DNS
        cidr: 8.8.8.8/32
    ports:
    - protocol: UDP
      port: 53
```

#### 默认策略配置
```yaml
# 默认拒绝所有入站流量
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-ingress
spec:
  podSelector: {}  # 选择所有Pod
  policyTypes:
  - Ingress
  # 不定义ingress规则，即拒绝所有入站流量

---
# 默认拒绝所有出站流量
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-egress
spec:
  podSelector: {}
  policyTypes:
  - Egress
  # 不定义egress规则，即拒绝所有出站流量

---
# 默认允许所有流量（取消默认拒绝）
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-all
spec:
  podSelector: {}
  policyTypes:
  - Ingress
  - Egress
  ingress:
  - {}
  egress:
  - {}
```

#### NetworkPolicy实践测试
