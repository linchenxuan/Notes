## Week 14：持久化存储PV和PVC

### 💾 Day 1-2：PV和PVC基础概念

#### PV（PersistentVolume）配置学习
```yaml
# 我的PV学习笔记
# 1. 静态PV配置
apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv-static
  labels:
    type: local
    env: test
spec:
  # 1. 容量
  capacity:
    storage: 5Gi
  
  # 2. 访问模式
  accessModes:
    - ReadWriteOnce      # 单节点读写
    # - ReadOnlyMany     # 多节点只读
    # - ReadWriteMany    # 多节点读写
    # - ReadWriteOncePod # 单Pod读写（1.22+）
  
  # 3. 回收策略
  persistentVolumeReclaimPolicy: Retain  # Retain, Delete, Recycle
  
  # 4. 存储类型
  storageClassName: manual  # 指定StorageClass
  
  # 5. 挂载选项
  mountOptions:
    - hard
    - intr
    - nolock
  
  # 6. 存储后端配置
  hostPath:  # 本地存储
    path: /mnt/data
    type: DirectoryOrCreate
  
  # 其他存储类型：
  # nfs:
  #   server: 192.168.1.100
  #   path: /exports/data
  # 
  # csi:
  #   driver: example.com/driver
  #   volumeHandle: volume-id
```

#### PVC（PersistentVolumeClaim）配置学习
```yaml
# 我的PVC学习笔记
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: pvc-test
  namespace: default
spec:
  # 1. 访问模式（必须与PV匹配）
  accessModes:
    - ReadWriteOnce
  
  # 2. 资源请求
  resources:
    requests:
      storage: 3Gi
    # limits:
    #   storage: 10Gi
  
  # 3. 存储类（可选）
  storageClassName: manual
  
  # 4. 选择器（可选）
  selector:
    matchLabels:
      type: local
      env: test
    matchExpressions:
      - key: type
        operator: In
        values: ["local", "nfs"]
  
  # 5. 卷模式（可选）
  volumeMode: Filesystem  # Filesystem, Block
  
  # 6. 数据源（可选，用于克隆或恢复）
  # dataSource:
  #   name: snapshot-name
  #   kind: VolumeSnapshot
  #   apiGroup: snapshot.storage.k8s.io
```

#### PV和PVC实践测试
```bash
# 我的PV和PVC实践记录

# 1. 创建静态PV
kubectl apply -f pv-static.yaml

# 2. 创建PVC
kubectl apply -f pvc-test.yaml

# 3. 查看PV和PVC状态
kubectl get pv
kubectl get pvc

# 4. 创建测试Pod使用PVC
kubectl apply -f pod-with-pvc.yaml

# 5. 验证存储挂载
kubectl exec -it pod-with-pvc -- sh -c 'echo "test data" > /data/test.txt'
kubectl exec -it pod-with-pvc -- cat /data/test.txt

# 6. 删除Pod后验证数据持久性
kubectl delete pod pod-with-pvc
# 重新创建Pod
kubectl apply -f pod-with-pvc.yaml
# 验证数据是否还存在
kubectl exec -it pod-with-pvc -- cat /data/test.txt
```

#### Pod使用PVC配置
```yaml
# 我的Pod使用PVC配置
apiVersion: v1
kind: Pod
metadata:
  name: pod-with-pvc
spec:
  containers:
  - name: app
    image: nginx:1.23
    volumeMounts:
    - name: data-volume
      mountPath: /data
      subPath: nginx  # 子路径挂载
      readOnly: false
  
  volumes:
  - name: data-volume
    persistentVolumeClaim:
      claimName: pvc-test
      readOnly: false  # 是否只读挂载
```

**我的理解总结**：
- **PV**：集群级别的存储资源，由管理员创建
- **PVC**：命名空间级别的存储请求，由用户创建
- **绑定**：PVC会自动匹配符合条件的PV
- **生命周期**：PV和PVC有独立的生命周期

### 🔄 Day 3-4：StorageClass动态存储

#### StorageClass配置学习
```yaml
# 我的StorageClass学习笔记
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: fast-storage
  annotations:
    storageclass.kubernetes.io/is-default-class: "true"  # 设为默认
provisioner: kubernetes.io/aws-ebs  # 存储供应商
parameters:  # 供应商特定参数
  type: gp3
  fsType: ext4
  iopsPerGB: "10"
  encrypted: "true"
  
# 回收策略
reclaimPolicy: Delete  # Delete, Retain

# 挂载选项
mountOptions:
  - debug
  - noatime

# 卷绑定模式
volumeBindingMode: Immediate  # Immediate, WaitForFirstConsumer

# 允许扩展
allowVolumeExpansion: true
```

```yaml
# 我的高级ConfigMap学习
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config-advanced
  namespace: default
data:

      port: 8080

    user nginx;
    worker_processes auto;
    error_log /var/log/nginx/error.log;
    pid /run/nginx.pid;
    
    events {
      worker_connections 1024;
    }
    
    http {
      log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';
      
      access_log /var/log/nginx/access.log main;
      
      sendfile on;
      tcp_nopush on;
      tcp_nodelay on;
      keepalive_timeout 65;
      types_hash_max_size 2048;

      default_type application/octet-stream;

        listen 80 default_server;
        listen [::]:80 default_server;
        server_name _;
        root /usr/share/nginx/html;
        
        location / {
          try_files $uri $uri/ =404;
        }
        
        location /health {
          access_log off;
          return 200 "healthy\n";
          add_header Content-Type text/plain;
        }
      }
    }
  
  # 3. 环境变量
  app.env: |
    APP_NAME=MyApplication
    APP_VERSION=1.0.0
    FEATURE_FLAGS=feature1,feature2,feature3
    
  # 4. 脚本文件
  init.sh: |
    #!/bin/bash
    set -e
    
    echo "Initializing application..."
    
    # 等待数据库就绪
    until nc -z postgres 5432; do
      echo "Waiting for postgres..."
      sleep 1
    done
    
    # 运行数据库迁移
    echo "Running database migrations..."
    java -jar /app/migrate.jar
    
    # 启动应用

    exec java -jar /app/application.jar
```

#### ConfigMap使用技巧
```yaml
# 我的ConfigMap使用技巧
apiVersion: v1
kind: Pod
metadata:
  name: configmap-demo
spec:
  initContainers:
  - name: config-validator
    image: busybox
    command: ["/bin/sh"]
    args:
    - -c
    - |
      echo "Validating configuration..."
      # 验证配置文件语法
      if ! grep -q "server:" /config/application.yml; then
        echo "Invalid application configuration"
        exit 1
      fi
      echo "Configuration validation passed"
    volumeMounts:
    - name: config-volume
      mountPath: /config
      readOnly: true
  
  containers:
  - name: app
    image: myapp:latest
    env:
    # 从ConfigMap导入所有键值对
    - name: APP_CONFIG_PATH
      value: /etc/app/config
    envFrom:
    - configMapRef:
        name: app-config-advanced
        optional: false  # 必须存在
    
    # 单独导入特定键
    - name: APP_NAME
      valueFrom:
        configMapKeyRef:
          name: app-config-advanced
          key: app.env
          optional: true  # 可选
    
    volumeMounts:
    # 整个ConfigMap挂载为目录
    - name: config-volume
      mountPath: /etc/app/config
      readOnly: true
    
    # 单个文件挂载
    - name: nginx-config
      mountPath: /etc/nginx/nginx.conf
      subPath: nginx.conf
    
    # 子路径挂载
    - name: app-config
      mountPath: /app/config/application.yml
      subPath: application.yml
    
    # 环境变量文件挂载
    - name: app-env
      mountPath: /app/.env
      subPath: app.env
  
  volumes:
  - name: config-volume
    configMap:
      name: app-config-advanced
      defaultMode: 0644  # 文件权限
      optional: false
  
  - name: nginx-config
    configMap:
      name: app-config-advanced
      items:
      - key: nginx.conf
        path: nginx.conf
        mode: 0644
  
  - name: app-config
    configMap:
      name: app-config-advanced
      items:
      - key: application.yml
        path: application.yml
  
  - name: app-env
    configMap:
      name: app-config-advanced
      items:
      - key: app.env
        path: .env
        mode: 0600  # 更严格的权限
```

#### Secret使用安全实践
```yaml
# 我的Secret安全使用配置
apiVersion: v1
kind: Pod
metadata:
  name: secret-security-demo
spec:
  containers:
  - name: secure-app
    image: myapp:latest
    env:
    # 1. 环境变量方式（不推荐，容易泄露）
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: app-secrets
          key: db-password
    
    volumeMounts:
    # 2. 文件挂载方式（推荐）
    - name: secrets-volume
      mountPath: /etc/secrets
      readOnly: true
    
    # 3. 单个文件挂载
    - name: tls-cert
      mountPath: /etc/ssl/certs/tls.crt
      subPath: tls.crt
      readOnly: true
    
    - name: tls-key
      mountPath: /etc/ssl/private/tls.key
      subPath: tls.key
      readOnly: true
      mode: 0400  # 严格权限
    
    # 4. 内存文件系统挂载（最安全）
    - name: memory-secrets
      mountPath: /run/secrets
      readOnly: true
    
    # 5. 安全上下文
    securityContext:
      runAsNonRoot: true
      runAsUser: 1000
      runAsGroup: 1000
      fsGroup: 1000
      capabilities:
        drop:
        - ALL
      readOnlyRootFilesystem: true
    
    resources:
      requests:
        memory: "128Mi"
        cpu: "100m"
      limits:
        memory: "256Mi"
        cpu: "200m"
  
  volumes:
  - name: secrets-volume
    secret:
      secretName: app-secrets
      defaultMode: 0400  # 只读权限
      optional: false
  
  - name: tls-cert
    secret:
      secretName: tls-secret
      items:
      - key: tls.crt
        path: tls.crt
        mode: 0444
  
  - name: tls-key
    secret:
      secretName: tls-secret
      items:
      - key: tls.key
        path: tls.key
        mode: 0400
  
  - name: memory-secrets
    secret:
      secretName: app-secrets
      defaultMode: 0400
  
  # 6. 镜像拉取Secret
  imagePullSecrets:
  - name: docker-registry-secret
  
  # 7. Service Account（自动挂载token）
  serviceAccountName: secure-service-account
  automountServiceAccountToken: false  # 手动控制是否挂载
```

```yaml
apiVersion: apps/v1

          mountPath: /etc/app/config
          readOnly: true

        - name: config-reload-script
          mountPath: /scripts/reload-config.sh
          subPath: reload-config.sh
        
        lifecycle:
          postStart:
            exec:
              command: ["/bin/sh", "-c", "/scripts/reload-config.sh &"]
      
      volumes:
      - name: config-volume
        configMap:
          name: app-config
          optional: false
      
      - name: config-reload-script
        configMap:
          name: config-reload-script
          defaultMode: 0755

---
# 配置重载脚本
apiVersion: v1
kind: ConfigMap
metadata:
  name: config-reload-script
data:
  reload-config.sh: |
    #!/bin/bash
    
    echo "Starting configuration watcher..."
    
    while true; do
      # 检查配置文件是否有变化
      if [ -f /etc/app/config/application.yml ]; then
        current_checksum=$(md5sum /etc/app/config/application.yml | cut -d' ' -f1)
        
        if [ "$current_checksum" != "$last_checksum" ]; then
          echo "Configuration changed, reloading..."
          
          # 发送信号给应用进程重载配置
          if [ -f /var/run/app.pid ]; then
            kill -HUP $(cat /var/run/app.pid) 2>/dev/null || true
          fi
          
          # 或者重启应用（如果应用不支持热重载）
          # kill -TERM $(cat /var/run/app.pid) 2>/dev/null || true
          
          last_checksum=$current_checksum
          echo "Configuration reloaded successfully"
        fi
      fi
      
      sleep 30
    done
```
