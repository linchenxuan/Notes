# 5.1 游戏服务专项：长连接与会话保持

> 本节目标：解决 WebSocket/TCP 长连接在 K8s 中的挑战，实现优雅关闭和平滑迁移。

---

## 一、长连接在 K8s 中的挑战

### 问题

| 挑战 | 原因 |
|------|------|
| **负载不均** | Service 默认随机/轮询，新连接可能全到新 Pod |
| **滚动更新断连** | Pod 被终止时，已有 WebSocket 连接直接断开 |
| **扩容无效** | 长连接已建立，新 Pod 没有新连接进来 |
| **缩容丢连接** | Pod 被删除，上面的连接全部丢失 |

---

## 二、Session Affinity（会话保持）

### 配置

```yaml
apiVersion: v1
kind: Service
metadata:
  name: game-gate-svc
spec:
  selector:
    app: game-gate
  sessionAffinity: ClientIP
  sessionAffinityConfig:
    clientIP:
      timeoutSeconds: 3600    # 保持 1 小时
  ports:
  - port: 8080
    targetPort: 8080
```

**原理**：同一客户端 IP 的请求会被路由到同一个 Pod（kube-proxy 通过 iptables recent 模块实现）。

**局限**：
- 基于 Client IP，NAT 后多个客户端可能被识别为同一 IP
- 不适用于 Ingress（Ingress 有自己的 session affinity 配置）

## 三、优雅关闭机制

### 完整流程

```
1. kubectl 发起 Pod 终止
2. Pod 从 Service Endpoints 移除（不再接收新连接）
3. 执行 preStop hook
4. 发送 SIGTERM 信号给容器主进程
5. 等待 terminationGracePeriodSeconds（默认 30s）
6. 超时后发送 SIGKILL 强制终止
```

```
时间线：
├── Endpoints 移除 (立即)
├── preStop hook 执行
├── SIGTERM 发送
├── 等待连接排空... ─────────────────────┤
│                                         │
├─── terminationGracePeriodSeconds ───────┤
│                                         │
├── SIGKILL (超时强制终止)                  │
```

### Go 代码示例：优雅关闭 WebSocket 服务

```go
package main

import (
    "context"
    "log"
    "net/http"
    "os"
    "os/signal"
    "sync"
    "sync/atomic"
    "syscall"
    "time"

    "github.com/gorilla/websocket"
)

var (
    upgrader = websocket.Upgrader{
        CheckOrigin: func(r *http.Request) bool { return true },
    }
    connections sync.Map      // 活跃连接管理
    connCount   int64         // 连接计数
    isShutdown  atomic.Bool   // 关闭标记
)

func handleWebSocket(w http.ResponseWriter, r *http.Request) {
    // 拒绝新连接（如果正在关闭）
    if isShutdown.Load() {
        http.Error(w, "server shutting down", http.StatusServiceUnavailable)
        return
    }

    conn, err := upgrader.Upgrade(w, r, nil)
    if err != nil {
        return
    }

    id := atomic.AddInt64(&connCount, 1)
    connections.Store(id, conn)
    defer func() {
        connections.Delete(id)
        conn.Close()
    }()

    // 处理消息
    for {
        _, msg, err := conn.ReadMessage()
        if err != nil {
            break
        }
        conn.WriteMessage(websocket.TextMessage, msg) // echo
    }
}

func main() {
    server := &http.Server{Addr: ":8080"}
    http.HandleFunc("/ws", handleWebSocket)
    http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
        if isShutdown.Load() {
            w.WriteHeader(http.StatusServiceUnavailable)
            return
        }
        w.WriteHeader(http.StatusOK)
    })

    // 启动服务
    go func() {
        log.Println("Game Gate starting on :8080")
        if err := server.ListenAndServe(); err != http.ErrServerClosed {
            log.Fatal(err)
        }
    }()

    // 监听信号
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGTERM, syscall.SIGINT)
    <-quit

    log.Println("Received shutdown signal")
    isShutdown.Store(true)

    // 1. 通知所有客户端即将关闭，让客户端重连到其他 Pod
    connections.Range(func(key, value interface{}) bool {
        conn := value.(*websocket.Conn)
        msg := websocket.FormatCloseMessage(
            websocket.CloseGoingAway,
            "server shutting down, please reconnect",
        )
        conn.WriteControl(websocket.CloseMessage, msg, time.Now().Add(5*time.Second))
        return true
    })

    // 2. 等待连接排空（最多等待 25 秒，留 5 秒给 SIGKILL）
    deadline := time.After(25 * time.Second)
    ticker := time.NewTicker(1 * time.Second)
    defer ticker.Stop()

    for {
        select {
        case <-deadline:
            log.Println("Deadline reached, force closing remaining connections")
            goto shutdown
        case <-ticker.C:
            count := 0
            connections.Range(func(_, _ interface{}) bool {
                count++
                return true
            })
            log.Printf("Waiting for %d connections to drain...", count)
            if count == 0 {
                log.Println("All connections drained")
                goto shutdown
            }
        }
    }

shutdown:
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
    server.Shutdown(ctx)
    log.Println("Server gracefully stopped")
}
```

### Deployment YAML

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game-gate
spec:
  replicas: 3
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0      # 确保不中断
  selector:
    matchLabels:
      app: game-gate
  template:
    metadata:
      labels:
        app: game-gate
    spec:
      terminationGracePeriodSeconds: 60    # 给足排空时间
      containers:
      - name: gate
        image: game-gate:v1.0
        ports:
        - containerPort: 8080
        lifecycle:
          preStop:
            exec:
              command:
              - /bin/sh
              - -c
              - |
                echo "preStop: waiting for endpoints to deregister..."
                sleep 5
                # sleep 给 kube-proxy 时间更新 iptables，
                # 确保新请求不再路由到此 Pod
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          periodSeconds: 5
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 10
---
apiVersion: v1
kind: Service
metadata:
  name: game-gate-svc
spec:
  selector:
    app: game-gate
  sessionAffinity: ClientIP
  sessionAffinityConfig:
    clientIP:
      timeoutSeconds: 3600
  ports:
  - port: 8080
    targetPort: 8080
```

## 四、PodDisruptionBudget（PDB）

PDB 保证在自愿中断（如滚动更新、节点维护）时，至少有一定数量的 Pod 可用：

```yaml
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata:
  name: game-gate-pdb
spec:
  minAvailable: 2          # 至少保持 2 个 Pod 可用
  # 或 maxUnavailable: 1   # 最多 1 个 Pod 不可用
  selector:
    matchLabels:
      app: game-gate
```

## 五、测试滚动更新连接迁移

```bash
# 终端 1：启动 WebSocket 客户端（可用 wscat）
# npm install -g wscat
wscat -c ws://localhost:8080/ws
# 保持连接

# 终端 2：执行滚动更新
kubectl set image deployment/game-gate gate=game-gate:v2.0

# 观察终端 1：
# 应收到 close frame (1001 Going Away) + "please reconnect" 消息
# 客户端可自动重连到新 Pod

# 终端 3：监控 Pod 变化
kubectl get pods -w -l app=game-gate
# 应看到新 Pod 逐步创建，旧 Pod 在排空后终止
```
