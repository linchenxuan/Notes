# 5.2 游戏服务专项：专用游戏服务器（DS）编排

> 本节目标：使用 Agones 实现游戏专用服务器（Dedicated Server）的编排和自动扩缩。

---

## 一、Agones 概述

### 什么是 Agones

Agones 是 Google 开源的 K8s 扩展，专为**多人游戏专用服务器**设计。解决传统 K8s 不擅长的问题：

- 游戏服务器需要**直接暴露端口**（UDP/TCP），不走 Service 负载均衡
- 每个游戏服务器是**独立的有状态实例**（一局游戏 = 一个进程）
- 需要**分配机制**（匹配系统请求一个空闲服务器）
- 生命周期由**游戏逻辑控制**（一局结束后关闭）

### Agones vs 普通 K8s

| 特性 | 普通 K8s | Agones |
|------|---------|--------|
| 端口暴露 | Service 负载均衡 | 每个 Pod 独立 hostPort |
| 生命周期 | K8s 管理 | 游戏逻辑通过 SDK 控制 |
| 扩缩容 | HPA（基于指标） | FleetAutoscaler（基于 Ready 缓冲池） |
| 分配 | 无 | GameServerAllocation（分配空闲服务器） |

---

## 二、核心 CRD

### GameServer

最小单元，代表一个游戏服务器实例。

生命周期：
```
Creating → Starting → Ready → Allocated → Shutdown
                        ↑         │
                        └─────────┘ (可选：返回 Ready 重新使用)
```

| 状态 | 含义 |
|------|------|
| Creating | Pod 正在创建 |
| Starting | Pod 已创建，游戏进程启动中 |
| Ready | 空闲可用，等待玩家 |
| Allocated | 已分配给一局游戏 |
| Shutdown | 游戏结束，准备销毁 |

### Fleet

管理一组 GameServer（类似 Deployment 管理 Pod）。

### FleetAutoscaler

自动扩缩 Fleet 中 GameServer 的数量。

### GameServerAllocation

请求分配一个 Ready 状态的 GameServer。

## 四、实操：简单 UDP 游戏服务器

### Go UDP 服务器代码

```go
package main

import (
    "fmt"
    "log"
    "net"
    "os"
    "strings"
    "time"

    sdk "agones.dev/agones/sdks/go"
)

func main() {
    // 初始化 Agones SDK
    s, err := sdk.NewSDK()
    if err != nil {
        log.Fatalf("Could not connect to Agones SDK: %v", err)
    }

    // 健康检查（定期向 Agones 报告存活）
    go func() {
        tick := time.Tick(2 * time.Second)
        for range tick {
            s.Health()
        }
    }()

    // 监听 UDP
    port := os.Getenv("PORT")
    if port == "" {
        port = "7654"
    }

    addr, _ := net.ResolveUDPAddr("udp", ":"+port)
    conn, err := net.ListenUDP("udp", addr)
    if err != nil {
        log.Fatalf("Could not listen on UDP: %v", err)
    }
    defer conn.Close()
    log.Printf("Game server listening on UDP :%s", port)

    // 标记为 Ready（告诉 Agones 可以接收玩家了）
    if err := s.Ready(); err != nil {
        log.Fatalf("Could not mark as Ready: %v", err)
    }
    log.Println("Marked as Ready")

    // 处理消息
    buf := make([]byte, 1024)
    for {
        n, remote, err := conn.ReadFromUDP(buf)
        if err != nil {
            continue
        }
        msg := strings.TrimSpace(string(buf[:n]))
        log.Printf("Received from %s: %s", remote, msg)

        switch msg {
        case "PING":
            conn.WriteToUDP([]byte("PONG\n"), remote)
        case "EXIT":
            // 游戏结束，标记 Shutdown
            log.Println("Game over, shutting down")
            s.Shutdown()
            os.Exit(0)
        default:
            response := fmt.Sprintf("Echo: %s\n", msg)
            conn.WriteToUDP([]byte(response), remote)
        }
    }
}
```

### 单个 GameServer YAML

```yaml
# gameserver.yaml
apiVersion: agones.dev/v1
kind: GameServer
metadata:
  generateName: game-room-    # 自动生成名称
spec:
  ports:
  - name: default
    portPolicy: Dynamic       # 动态分配 hostPort
    containerPort: 7654
    protocol: UDP
  template:
    spec:
      containers:
      - name: game-room
        image: your-registry/game-room:v1.0
        resources:
          requests:
            cpu: "100m"
            memory: "64Mi"
          limits:
            cpu: "500m"
            memory: "128Mi"
```

```bash
kubectl apply -f gameserver.yaml

# 查看 GameServer
kubectl get gs
# NAME             STATE   ADDRESS       PORT   NODE      AGE
# game-room-abc12  Ready   2.3.80.90   7890   node-1    10s

# 测试连接（UDP）
nc -u 2.3.80.90 7890
PING
# PONG
```

## 五、实操：Fleet 和 FleetAutoscaler

### Fleet YAML

```yaml
# fleet.yaml
apiVersion: agones.dev/v1
kind: Fleet
metadata:
  name: game-room-fleet
spec:
  replicas: 5               # 初始 5 个 GameServer
  template:
    spec:
      ports:
      - name: default
        portPolicy: Dynamic
        containerPort: 7654
        protocol: UDP
      template:
        spec:
          containers:
          - name: game-room
            image: your-registry/game-room:v1.0
            resources:
              requests:
                cpu: "100m"
                memory: "64Mi"
              limits:
                cpu: "500m"
                memory: "128Mi"
```

### FleetAutoscaler YAML（Buffer 策略）

```yaml
# fleet-autoscaler.yaml
apiVersion: autoscaling.agones.dev/v1
kind: FleetAutoscaler
metadata:
  name: game-room-autoscaler
spec:
  fleetName: game-room-fleet
  policy:
    type: Buffer
    buffer:
      bufferSize: 3          # 始终保持 3 个 Ready 状态的 GameServer
      minReplicas: 3
      maxReplicas: 50
```

**Buffer 策略**：始终维持一个 Ready 状态的缓冲池。当 Ready 的 GameServer 被分配（变为 Allocated），自动创建新的补充。

```bash
kubectl apply -f fleet.yaml
kubectl apply -f fleet-autoscaler.yaml

# 查看 Fleet
kubectl get fleet
# NAME              SCHEDULING   DESIRED   CURRENT   ALLOCATED   READY
# game-room-fleet   Packed       5         5         0           5

# 查看所有 GameServer

# NAME                       STATE   ADDRESS       PORT
# game-room-fleet-abc12      Ready   2.3.80.90   7890
# game-room-fleet-def34      Ready   2.3.80.90   7891

# game-room-fleet-mno90      Ready   2.3.80.90   7894
```

## 六、实操：GameServerAllocation（模拟匹配）

```yaml
# allocation.yaml
apiVersion: allocation.agones.dev/v1
kind: GameServerAllocation
spec:
  required:
    matchLabels:
      agones.dev/fleet: game-room-fleet
```

```bash
# 分配一个 GameServer
kubectl create -f allocation.yaml -o yaml
# status:
#   state: Allocated
#   gameServerName: game-room-fleet-abc12
#   address: 2.3.80.90
#   ports:
#   - name: default
#     port: 7890

# 查看状态变化
kubectl get gs
# NAME                       STATE       ADDRESS       PORT
# game-room-fleet-abc12      Allocated   2.3.80.90   7890  ← 已分配
# game-room-fleet-def34      Ready       2.3.80.90   7891
# ...
# game-room-fleet-new01      Ready       2.3.80.90   7895  ← 自动新建补充

# 持续分配，观察自动扩容
for i in $(seq 1 5); do
  kubectl create -f allocation.yaml -o jsonpath='{.status.gameServerName}{"\n"}'
done
# game-room-fleet-def34
# game-room-fleet-ghi56
# ...

# Fleet 自动扩容
kubectl get fleet
# NAME              DESIRED   CURRENT   ALLOCATED   READY
# game-room-fleet   8         8         5           3    ← Ready 保持 3 个缓冲
```

### 匹配系统集成流程

```
1. 玩家请求匹配 → 匹配服务凑齐玩家
2. 匹配服务 → 调用 K8s API 创建 GameServerAllocation
3. Agones 返回 Allocated GameServer 的地址和端口
4. 匹配服务 → 将地址端口发给客户端
5. 客户端直连 GameServer（UDP/TCP）
6. 游戏结束 → GameServer 调用 SDK.Shutdown()
7. Agones 销毁该 GameServer，Fleet 自动补充新的
```
