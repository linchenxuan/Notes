
## 二、Pod Security Standards（PSS）

### 三个安全级别

| 级别 | 限制程度 | 适用场景 |
|------|----------|----------|
| **Privileged** | 无限制 | 系统组件（如 CNI 插件） |
| **Baseline** | 最低安全标准 | 一般应用 |
| **Restricted** | 最严格 | 安全敏感应用 |

### Baseline 禁止的主要行为

- 特权容器（privileged: true）
- hostNetwork / hostPID / hostIPC
- 危险的 capabilities（NET_RAW 除外）
- hostPath volumes

### Restricted 额外要求

- 必须以非 root 运行
- 必须设置 readOnlyRootFilesystem
- 不允许 privilege escalation
- 必须 drop ALL capabilities

## 三、Pod Security Admission（PSA）

### 配置方式

通过 Namespace label 控制：

```bash
# 格式：pod-security.kubernetes.io/<mode>: <level>
# mode: enforce（拒绝）/ audit（审计日志）/ warn（警告）

# 配置 game-prod 命名空间：强制执行 baseline 级别
kubectl label namespace game-prod \
  pod-security.kubernetes.io/enforce=baseline \
  pod-security.kubernetes.io/enforce-version=latest \
  pod-security.kubernetes.io/audit=restricted \
  pod-security.kubernetes.io/warn=restricted
```

### 完整实操

```bash
# 1. 创建命名空间并配置安全策略
kubectl create namespace secure-game

kubectl label namespace secure-game \
  pod-security.kubernetes.io/enforce=baseline \
  pod-security.kubernetes.io/warn=restricted

# 2. 尝试部署特权容器 — 应被拒绝
cat <<EOF | kubectl apply -n secure-game -f -
apiVersion: v1
kind: Pod
metadata:
  name: privileged-pod
spec:
  containers:
  - name: test
    image: nginx:1.25
    securityContext:
      privileged: true
EOF
# Error from server (Forbidden): pods "privileged-pod" is forbidden:
# violates PodSecurity "baseline:latest":
# privileged (container "test" must not set securityContext.privileged=true)
# ✅ 被拒绝

# 3. 部署合规 Pod — 应成功
cat <<EOF | kubectl apply -n secure-game -f -
apiVersion: v1
kind: Pod
metadata:
  name: safe-pod
spec:
  containers:
  - name: test
    image: nginx:1.25
    securityContext:
      allowPrivilegeEscalation: false
      runAsNonRoot: true
      runAsUser: 1000
      capabilities:
        drop: ["ALL"]
      seccompProfile:
        type: RuntimeDefault
EOF
# pod/safe-pod created  ✅

# 4. Warning（restricted 模式是 warn，会显示警告但不阻止）
cat <<EOF | kubectl apply -n secure-game -f -
apiVersion: v1
kind: Pod
metadata:
  name: baseline-pod
spec:
  containers:
  - name: test
    image: nginx:1.25
EOF
# Warning: would violate PodSecurity "restricted:latest": ...
# pod/baseline-pod created（warn 模式不阻止）
```

## 四、镜像安全扫描：Trivy

### 安装

```bash
# Linux
sudo apt-get install wget apt-transport-https gnupg lsb-release
wget -qO - https://aquasecurity.github.io/trivy-repo/deb/public.key | sudo apt-key add -
echo deb https://aquasecurity.github.io/trivy-repo/deb $(lsb_release -sc) main | sudo tee /etc/apt/sources.list.d/trivy.list
sudo apt-get update
sudo apt-get install trivy

# 或直接下载二进制
curl -sfL https://raw.githubusercontent.com/aquasecurity/trivy/main/contrib/install.sh | sh -s -- -b /usr/local/bin
```

### 扫描镜像

```bash
# 扫描游戏服务镜像
trivy image game-gate:v1.0

# 输出示例：
# game-gate:v1.0 (alpine 3.19)
# =============================
# Total: 5 (UNKNOWN: 0, LOW: 2, MEDIUM: 1, HIGH: 1, CRITICAL: 1)
#
# ┌──────────────┬────────────────┬──────────┬───────────────┬──────────────────┐
# │   Library    │ Vulnerability  │ Severity │    Installed   │     Fixed        │
# ├──────────────┼────────────────┼──────────┼───────────────┼──────────────────┤
# │ libcrypto3   │ CVE-2024-xxxx  │ CRITICAL │ 3.1.4-r0      │ 3.1.4-r1         │
# │ libssl3      │ CVE-2024-xxxx  │ HIGH     │ 3.1.4-r0      │ 3.1.4-r1         │
# │ busybox      │ CVE-2024-xxxx  │ MEDIUM   │ 1.36.1-r2     │ 1.36.1-r3        │
# └──────────────┴────────────────┴──────────┴───────────────┴──────────────────┘

# 只显示 HIGH 和 CRITICAL
trivy image --severity HIGH,CRITICAL game-gate:v1.0

# 扫描并以 JSON 输出（集成到 CI）
trivy image -f json -o results.json game-gate:v1.0

# 扫描失败退出码（CI 中阻断构建）
trivy image --exit-code 1 --severity CRITICAL game-gate:v1.0
```

### 修复漏洞

```dockerfile
# 更新基础镜像
FROM alpine:3.19    # → FROM alpine:3.20

# 在构建时更新包
RUN apk --no-cache upgrade

# 修复后重新扫描验证
trivy image game-gate:v1.1
# Total: 0 (CRITICAL: 0, HIGH: 0)  ✅
```

## 五、Kyverno 策略引擎

### 安装

```bash
helm repo add kyverno https://kyverno.github.io/kyverno/
helm install kyverno kyverno/kyverno -n kyverno --create-namespace
```

### 策略示例

#### 禁止使用 latest tag

```yaml
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: disallow-latest-tag
spec:
  validationFailureAction: Enforce
  rules:
  - name: validate-image-tag
    match:
      any:
      - resources:
          kinds:
          - Pod
    validate:
      message: "Using 'latest' image tag is not allowed. Please specify a version."
      pattern:
        spec:
          containers:
          - image: "!*:latest & !*"    # 禁止 latest 和无 tag
```

#### 强制设置资源限制

```yaml
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: require-resource-limits
spec:
  validationFailureAction: Enforce
  rules:
  - name: validate-resources
    match:
      any:
      - resources:
          kinds:
          - Pod
    validate:
      message: "CPU and memory limits are required."
      pattern:
        spec:
          containers:
          - resources:
              limits:
                cpu: "?*"
                memory: "?*"
```

## 六、安全最佳实践清单

```yaml
# 安全的 Pod 模板
spec:
  containers:
  - name: app
    image: game-gate:v1.0.0       # ✅ 明确版本，不用 latest
    securityContext:
      runAsNonRoot: true           # ✅ 非 root 运行
      runAsUser: 1000
      readOnlyRootFilesystem: true # ✅ 只读文件系统
      allowPrivilegeEscalation: false  # ✅ 禁止提权
      capabilities:
        drop: ["ALL"]              # ✅ 删除所有 capabilities
      seccompProfile:
        type: RuntimeDefault       # ✅ 启用 seccomp
    resources:
      limits:                      # ✅ 设置资源限制
        cpu: "500m"
        memory: "256Mi"
  automountServiceAccountToken: false  # ✅ 不需要时禁用 SA token
```
