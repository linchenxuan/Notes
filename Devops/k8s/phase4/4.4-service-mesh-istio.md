# 4.4 服务网格 — Istio（选学）

> 本节目标：理解服务网格概念，掌握 Istio 流量管理和灰度发布。

---

## 一、服务网格概念

### 解决的问题

微服务架构中，服务间通信面临的挑战：
- 流量管理（负载均衡、灰度发布、熔断）
- 安全（mTLS 加密、认证授权）
- 可观测性（链路追踪、流量监控）

服务网格将这些**基础设施关注点**从业务代码中剥离，下沉到基础设施层。

### Istio 架构

```
┌──────────────────────────────────────┐
│          Control Plane                │
│  ┌──────────────────────────────┐    │
│  │          istiod              │    │
│  │  (Pilot + Citadel + Galley) │    │
│  └──────────────┬───────────────┘    │
└─────────────────┼────────────────────┘
                  │ xDS 配置下发
┌─────────────────┼────────────────────┐
│  Data Plane     │                    │
│  ┌─────────┐   │   ┌─────────┐      │
│  │  Pod A  │   │   │  Pod B  │      │
│  │┌──────┐ │   │   │┌──────┐ │      │
│  ││App   │ │   │   ││App   │ │      │
│  │└──┬───┘ │   │   │└──┬───┘ │      │
│  │┌──┴───┐ │   │   │┌──┴───┐ │      │
│  ││Envoy │←┼───┼───┼│Envoy │ │      │
│  ││Sidecar│ │       ││Sidecar│ │      │
│  │└──────┘ │       │└──────┘ │      │
│  └─────────┘       └─────────┘      │
└──────────────────────────────────────┘
```

**Sidecar 模式**：每个 Pod 旁边注入一个 Envoy 代理容器，所有进出流量都经过 Envoy，对应用透明。

## 二、安装 Istio

```bash
# 1. 下载 istioctl
curl -L https://istio.io/downloadIstio | sh -
cd istio-*
export PATH=$PWD/bin:$PATH

# 2. 安装 Istio（demo profile，含所有组件）
istioctl install --set profile=demo -y

# 3. 验证
istioctl verify-install
kubectl get pods -n istio-system
# NAME                                    READY   STATUS
# istiod-xxx                              1/1     Running
# istio-ingressgateway-xxx                1/1     Running

# 4. 开启命名空间自动注入
kubectl label namespace game-dev istio-injection=enabled

# 5. 重启已有 Pod 使 sidecar 注入生效
kubectl rollout restart deployment -n game-dev
```

## 三、核心流量管理资源

### VirtualService（流量路由）

定义流量如何路由到目标服务。

### DestinationRule（目标规则）

定义到达目标服务后的策略（负载均衡、连接池、子集等）。

### Gateway（入口网关）

管理进入网格的外部流量。

## 四、完整实操：灰度发布

### 步骤 1：部署 v1 和 v2 两个版本

```yaml
# game-gate-v1v2.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game-gate-v1
  namespace: game-dev
spec:
  replicas: 3
  selector:
    matchLabels:
      app: game-gate
      version: v1
  template:
    metadata:
      labels:
        app: game-gate
        version: v1
    spec:
      containers:
      - name: gate
        image: nginx:1.25
        ports:
        - containerPort: 80
        command: ["/bin/sh", "-c"]
        args:
        - echo 'Game Gate v1' > /usr/share/nginx/html/index.html && nginx -g 'daemon off;'
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game-gate-v2
  namespace: game-dev
spec:
  replicas: 1
  selector:
    matchLabels:
      app: game-gate
      version: v2
  template:
    metadata:
      labels:
        app: game-gate
        version: v2
    spec:
      containers:
      - name: gate
        image: nginx:1.26
        ports:
        - containerPort: 80
        command: ["/bin/sh", "-c"]
        args:
        - echo 'Game Gate v2' > /usr/share/nginx/html/index.html && nginx -g 'daemon off;'
---
apiVersion: v1
kind: Service
metadata:
  name: game-gate-svc
  namespace: game-dev
spec:
  selector:
    app: game-gate     # 同时选中 v1 和 v2
  ports:
  - port: 80
```

### 步骤 2：配置 DestinationRule（定义子集）

```yaml
# destination-rule.yaml
apiVersion: networking.istio.io/v1beta1
kind: DestinationRule
metadata:
  name: game-gate-dr
  namespace: game-dev
spec:
  host: game-gate-svc
  subsets:
  - name: v1
    labels:
      version: v1
  - name: v2
    labels:
      version: v2
```

### 步骤 3：配置 VirtualService（灰度比例）

```yaml
# virtual-service.yaml
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: game-gate-vs
  namespace: game-dev
spec:
  hosts:
  - game-gate-svc
  http:
  - route:
    - destination:
        host: game-gate-svc
        subset: v1
      weight: 90          # 90% 流量到 v1
    - destination:
        host: game-gate-svc
        subset: v2
      weight: 10          # 10% 流量到 v2
```

```bash
kubectl apply -f game-gate-v1v2.yaml
kubectl apply -f destination-rule.yaml
kubectl apply -f virtual-service.yaml
```

### 步骤 4：验证流量分配

```bash
# 持续请求并统计
kubectl run -n game-dev test-client --rm -it --image=busybox:1.36 -- sh
# 在容器内：
for i in $(seq 1 100); do
  wget -qO- http://game-gate-svc 2>/dev/null
done | sort | uniq -c
#  90 Game Gate v1
#  10 Game Gate v2
# ✅ 约 9:1 比例
```

### 逐步提升 v2 流量

```yaml
# 调整为 50/50
- destination:
    host: game-gate-svc
    subset: v1
  weight: 50
- destination:
    host: game-gate-svc
    subset: v2
  weight: 50

# 全量切换到 v2
- destination:
    host: game-gate-svc
    subset: v2
  weight: 100
```

## 五、故障注入

### 延迟注入

```yaml
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: game-gate-fault
  namespace: game-dev
spec:
  hosts:
  - game-gate-svc
  http:
  - fault:
      delay:
        percentage:
          value: 50        # 50% 请求注入延迟
        fixedDelay: 3s     # 延迟 3 秒
    route:
    - destination:
        host: game-gate-svc
```

### 错误注入

```yaml
  http:
  - fault:
      abort:
        percentage:
          value: 10        # 10% 请求返回错误
        httpStatus: 503
    route:
    - destination:
        host: game-gate-svc
```

## 六、Kiali — 服务拓扑可视化

```bash
# 安装 Kiali
kubectl apply -f https://raw.githubusercontent.com/istio/istio/release-1.20/samples/addons/kiali.yaml

kubectl port-forward svc/kiali -n istio-system 20001:20001
```

- 服务拓扑图（实时流量箭头）
- 流量健康状态
