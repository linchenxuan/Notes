# 4.2 CI/CD — GitOps 流水线

> 本节目标：理解 GitOps 理念，搭建 ArgoCD 实现自动化部署。

---

## 一、GitOps 理念

### 核心原则

1. **声明式**：所有基础设施和应用配置都以声明式代码存储
2. **版本化**：Git 是唯一的事实来源（Single Source of Truth）
3. **自动化**：变更自动应用，漂移自动修复
4. **可审计**：所有变更通过 Git 历史追溯

### GitOps vs 传统 CI/CD

```
传统 CI/CD（Push 模式）：
  代码 → CI 构建镜像 → CD 脚本 push 到集群（kubectl apply）
  缺点：CD 工具需要集群权限，不安全

GitOps（Pull 模式）：
  代码 → CI 构建镜像 → 更新 Git 仓库配置 → ArgoCD 从 Git pull 并同步
  优点：集群内组件主动 pull，更安全；Git 作为唯一来源
```

```
┌──────┐    ┌────────┐    ┌──────────┐    ┌─────────┐
│ 开发者 │───→│  Git   │───→│ ArgoCD  │───→│  K8s    │
│ push  │    │ 仓库   │    │ (pull)  │    │ 集群    │
└──────┘    └────────┘    └──────────┘    └─────────┘
     │           ▲
     │    ┌──────┴───┐
     └───→│   CI     │ 构建镜像 + 更新 values
          │(Actions) │
          └──────────┘
```

## 二、ArgoCD 核心概念

| 概念 | 说明 |
|------|------|
| **Application** | 定义 Git 源 + 目标集群/namespace + 同步策略 |
| **AppProject** | 应用分组，控制权限和目标 |
| **Sync** | 将 Git 中的期望状态同步到集群 |
| **Sync Status** | Synced（一致）/ OutOfSync（不一致） |
| **Health Status** | Healthy / Degraded / Progressing / Missing |

## 三、ArgoCD 安装

```bash
# 1. 创建命名空间
kubectl create namespace argocd

# 2. 安装 ArgoCD
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# 3. 等待就绪
kubectl wait --for=condition=available deployment/argocd-server -n argocd --timeout=300s

# 4. 获取初始密码
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
# 输出类似: Abc123Xyz

# 5. 访问 ArgoCD UI
# 方式 1：port-forward
kubectl port-forward svc/argocd-server -n argocd 8443:443
# 访问 https://localhost:8443，用户名 admin，密码为上面获取的

# 方式 2：改为 NodePort
kubectl patch svc argocd-server -n argocd -p '{"spec":{"type":"NodePort"}}'
```

### 安装 ArgoCD CLI

```bash
# Linux
curl -sSL -o argocd https://github.com/argoproj/argo-cd/releases/latest/download/argocd-linux-amd64
chmod +x argocd
sudo mv argocd /usr/local/bin/

# 登录
argocd login localhost:8443 --username admin --password <初始密码> --insecure

# 修改密码
argocd account update-password
```

## 四、完整实操

### Git 仓库结构

```
game-deploy/
├── charts/
│   └── game-server/           # Helm Chart（同 4.1 节创建的）
│       ├── Chart.yaml
│       ├── values.yaml
│       └── templates/
├── envs/
│   ├── dev/
│   │   └── values.yaml        # 开发环境配置
│   └── prod/
│       └── values.yaml        # 生产环境配置
└── README.md
```

### 创建 ArgoCD Application

方式 1：YAML 声明式

```yaml
# argocd-app.yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: game-server-dev
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/yourname/game-deploy.git
    targetRevision: main
    path: charts/game-server
    helm:
      valueFiles:
      - ../../envs/dev/values.yaml
  destination:
    server: https://kubernetes.default.svc
    namespace: game-dev
  syncPolicy:
    automated:                 # 自动同步
      prune: true              # 删除 Git 中不存在的资源
      selfHeal: true           # 自动修复漂移
    syncOptions:
    - CreateNamespace=true     # 自动创建命名空间
```

```bash
kubectl apply -f argocd-app.yaml
```

方式 2：CLI

```bash
argocd app create game-server-dev \
  --repo https://github.com/yourname/game-deploy.git \
  --path charts/game-server \
  --dest-server https://kubernetes.default.svc \
  --dest-namespace game-dev \
  --helm-set-file values=envs/dev/values.yaml \
  --sync-policy automated \
  --auto-prune \
  --self-heal
```

### 查看和操作

```bash
# 查看应用状态
argocd app get game-server-dev
# Name:               game-server-dev
# Server:             https://kubernetes.default.svc
# Namespace:          game-dev
# Sync Status:        Synced
# Health Status:      Healthy

# 手动同步
argocd app sync game-server-dev

# 查看同步差异
argocd app diff game-server-dev

# 查看历史
argocd app history game-server-dev
# ID  DATE                 REVISION
# 1   2024-01-15 10:00:00  abc123
# 2   2024-01-15 14:30:00  def456

# 回滚
argocd app rollback game-server-dev 1
```

## 五、CI 构建示例（GitHub Actions）

```yaml
# .github/workflows/ci.yaml
name: Build and Deploy

on:
  push:
    branches: [main]
    paths:
    - 'src/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4

    - name: Build Docker Image
      run: |
        docker build -t ghcr.io/${{ github.repository }}/game-gate:${{ github.sha }} .
        docker push ghcr.io/${{ github.repository }}/game-gate:${{ github.sha }}

    - name: Update Helm Values
      run: |
        cd envs/dev
        yq e '.image.tag = "${{ github.sha }}"' -i values.yaml

    - name: Commit and Push
      run: |
        git config user.name "CI Bot"
        git config user.email "ci@example.com"
        git add envs/dev/values.yaml
        git commit -m "ci: update dev image to ${{ github.sha }}"
        git push
```

**流程**：
1. 开发者推送代码
2. CI 构建新镜像，tag 为 commit SHA
3. CI 更新 Git 仓库中的 values.yaml（image.tag）
4. ArgoCD 检测到 Git 变更，自动同步部署
