# 4.3 可观测性 — 监控 / 日志 / 链路追踪

> 本节目标：搭建 Prometheus + Grafana 监控栈，暴露自定义指标，配置告警。

## 一、可观测性三大支柱

| 支柱 | 工具 | 回答的问题 |
|------|------|-----------|
| **Metrics（指标）** | Prometheus + Grafana | 系统发生了什么？趋势如何？ |
| **Logs（日志）** | Loki / EFK | 为什么发生？上下文是什么？ |
| **Traces（链路）** | Jaeger / OpenTelemetry | 请求经过了哪些服务？瓶颈在哪？ |

## 二、Prometheus

### 核心概念

```
┌──────────┐  scrape  ┌──────────────┐  query  ┌─────────┐
│ Targets  │ ←────── │  Prometheus  │ ←───── │ Grafana │
│(暴露/metrics)│        │  (存储+计算)  │        │ (展示)  │
└──────────┘         └──────┬───────┘        └─────────┘
                            │ alert
                     ┌──────┴───────┐
                     │ Alertmanager │
                     │  (告警通知)   │
                     └──────────────┘
```

### 指标类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **Counter** | 只增不减的计数器 | 请求总数、错误总数 |
| **Gauge** | 可增可减的值 | 在线人数、内存使用量 |
| **Histogram** | 分桶统计分布 | 请求延迟、匹配等待时间 |
| **Summary** | 百分位数 | 延迟 P50/P99 |

## 三、安装 kube-prometheus-stack

```bash
# 添加 Helm 仓库
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

# 安装（包含 Prometheus + Grafana + Alertmanager + Node Exporter 等）
helm install monitoring prometheus-community/kube-prometheus-stack \
  -n monitoring --create-namespace \
  --set grafana.adminPassword=admin123

# 等待就绪
kubectl get pods -n monitoring -w

# 访问 Grafana
kubectl port-forward svc/monitoring-grafana -n monitoring 3000:80
# 浏览器访问 http://localhost:3000
# 用户名: admin  密码: admin123

# 访问 Prometheus
kubectl port-forward svc/monitoring-kube-prometheus-prometheus -n monitoring 9090:9090
# 浏览器访问 http://localhost:9090

# 访问 Alertmanager
kubectl port-forward svc/monitoring-kube-prometheus-alertmanager -n monitoring 9093:9093
```

## 四、在游戏服务中暴露自定义 Metrics

### Go 代码示例

```go
package main

import (
    "math/rand"
    "net/http"
    "time"

    "github.com/prometheus/client_golang/prometheus"
    "github.com/prometheus/client_golang/prometheus/promhttp"
)

var (
    // Gauge: 当前在线人数
    onlinePlayers = prometheus.NewGauge(prometheus.GaugeOpts{
        Name: "game_online_players",
        Help: "Current number of online players",
    })

    // Counter: 匹配总次数
    matchTotal = prometheus.NewCounterVec(prometheus.CounterOpts{
        Name: "game_match_total",
        Help: "Total number of matches",
    }, []string{"mode"})  // 按模式区分：ranked, casual

    // Histogram: 匹配等待延迟
    matchLatency = prometheus.NewHistogram(prometheus.HistogramOpts{
        Name:    "game_match_latency_seconds",
        Help:    "Match waiting latency in seconds",
        Buckets: []float64{0.1, 0.5, 1, 2, 5, 10, 30},
    })

    // Histogram: 请求延迟
    requestDuration = prometheus.NewHistogramVec(prometheus.HistogramOpts{
        Name:    "game_request_duration_seconds",
        Help:    "Request duration in seconds",
        Buckets: prometheus.DefBuckets,
    }, []string{"handler", "method"})
)

func init() {
    prometheus.MustRegister(onlinePlayers, matchTotal, matchLatency, requestDuration)
}

func main() {
    // 模拟指标变化
    go func() {
        for {
            onlinePlayers.Set(float64(rand.Intn(5000) + 3000))
            matchTotal.WithLabelValues("ranked").Inc()
            matchTotal.WithLabelValues("casual").Inc()
            matchLatency.Observe(rand.Float64()*10 + 0.1)
            time.Sleep(time.Second)
        }
    }()

    // 暴露 /metrics 端点
    http.Handle("/metrics", promhttp.Handler())
    http.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(200)
    })
    http.ListenAndServe(":8080", nil)
}
```

### Deployment 添加 annotations

```yaml
spec:
  template:
    metadata:
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/port: "8080"
        prometheus.io/path: "/metrics"
```

### 或创建 ServiceMonitor（推荐）

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: game-server-monitor
  namespace: monitoring
  labels:
    release: monitoring     # 需匹配 Prometheus 的 serviceMonitorSelector
spec:
  namespaceSelector:
    matchNames:
    - game-dev
  selector:
    matchLabels:
      app: game-gate
  endpoints:
  - port: http
    interval: 15s
    path: /metrics
```

## 五、配置 Grafana Dashboard

### 使用 PromQL 查询

```promql
# 当前在线人数
game_online_players

# 每秒匹配数（5分钟平均）
rate(game_match_total[5m])

# 匹配延迟 P99
histogram_quantile(0.99, rate(game_match_latency_seconds_bucket[5m]))

# 匹配延迟 P50
histogram_quantile(0.50, rate(game_match_latency_seconds_bucket[5m]))

# 请求错误率
sum(rate(game_request_total{status=~"5.."}[5m])) / sum(rate(game_request_total[5m]))
```

### Dashboard 配置步骤

1. 登录 Grafana → 左侧 "+" → Dashboard → Add Panel
2. Data source 选择 Prometheus
3. 输入 PromQL 查询
4. 选择可视化类型（Graph、Gauge、Stat 等）
5. 配置标题、单位、阈值颜色
6. Save Dashboard

**推荐面板布局**：

```
┌──────────────────────────────────────────────┐
│ Game Server Dashboard                         │
├──────────┬───────────┬───────────┬───────────┤
│ 在线人数  │ QPS      │ 匹配延迟P99│ 错误率    │
│ (Stat)   │ (Stat)   │ (Gauge)   │ (Stat)    │
├──────────┴───────────┴───────────┴───────────┤
│ 在线人数趋势 (Time Series)                     │
├──────────────────────┬───────────────────────┤
│ 匹配延迟分布          │ 各模式匹配数           │
│ (Histogram)          │ (Time Series)         │
└──────────────────────┴───────────────────────┘
```

## 六、配置告警

### PrometheusRule

```yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: game-alerts
  namespace: monitoring
  labels:
    release: monitoring
spec:
  groups:
  - name: game-server
    rules:
    # 匹配延迟告警
    - alert: HighMatchLatency
      expr: histogram_quantile(0.99, rate(game_match_latency_seconds_bucket[5m])) > 0.5
      for: 2m
      labels:
        severity: warning
      annotations:
        summary: "匹配延迟过高"
        description: "P99 匹配延迟 {{ $value }}s 超过 500ms 阈值，持续 2 分钟"

    # 在线人数骤降告警
    - alert: OnlinePlayersDrop
      expr: game_online_players < 100
      for: 1m
      labels:
        severity: critical
      annotations:
        summary: "在线人数异常偏低"
        description: "当前在线人数 {{ $value }}，低于 100 人阈值"

    # Pod 重启告警
    - alert: PodCrashLooping
      expr: rate(kube_pod_container_status_restarts_total{namespace="game-dev"}[15m]) > 0
      for: 5m
      labels:
        severity: critical
      annotations:
        summary: "Pod 频繁重启"
        description: "{{ $labels.pod }} 在 15 分钟内发生重启"
```

```bash
kubectl apply -f game-alerts.yaml

# 验证规则已加载
kubectl port-forward svc/monitoring-kube-prometheus-prometheus -n monitoring 9090:9090
# 访问 http://localhost:9090/alerts 查看告警规则
```

### Alertmanager 配置（通知渠道）

```yaml
# alertmanager-config.yaml（通过 Helm values 配置）
alertmanager:
  config:
    route:
      receiver: 'webhook'
      group_wait: 30s
      group_interval: 5m
      repeat_interval: 1h
      routes:
      - match:
          severity: critical
        receiver: 'webhook'
    receivers:
    - name: 'webhook'
      webhook_configs:
      - url: 'http://your-alert-webhook:9093/alert'
```

## 七、日志与链路追踪简介

### 日志方案：Loki

```bash
# 安装 Loki + Promtail
helm install loki grafana/loki-stack \
  -n monitoring \
  --set promtail.enabled=true \
  --set loki.persistence.enabled=true
```

在 Grafana 中添加 Loki 数据源，即可在 Explore 中查询日志：
```
{namespace="game-dev", app="game-gate"} |= "error"
```

### 链路追踪：OpenTelemetry + Jaeger

适用于微服务间调用链分析，能追踪一个请求从 Gate → Match → GameRoom 的完整路径和耗时。
