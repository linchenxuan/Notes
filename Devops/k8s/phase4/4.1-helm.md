# 4.1 Helm — 包管理

> 本节目标：掌握 Helm Chart 开发，将游戏服务封装为可复用的 Chart。

---

## 一、核心概念

| 概念 | 说明 |
|------|------|
| **Chart** | Helm 的包格式，包含一组 K8s 资源模板 |
| **Release** | Chart 的一次安装实例（同一 Chart 可安装多次） |
| **Repository** | Chart 的存储仓库 |
| **values.yaml** | Chart 的配置参数文件 |

---

## 二、安装 Helm

```bash
# Linux
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# 验证
helm version
# version.BuildInfo{Version:"v3.14.0", ...}

# 添加常用仓库
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update
```

## 三、Chart 目录结构

```
game-chart/
├── Chart.yaml              # Chart 元信息
├── values.yaml             # 默认配置值
├── values-dev.yaml         # 开发环境覆盖
├── values-prod.yaml        # 生产环境覆盖
├── templates/
│   ├── _helpers.tpl        # 模板辅助函数
│   ├── deployment.yaml     # Deployment 模板
│   ├── service.yaml        # Service 模板
│   ├── configmap.yaml      # ConfigMap 模板
│   ├── secret.yaml         # Secret 模板
│   ├── hpa.yaml            # HPA 模板
│   ├── ingress.yaml        # Ingress 模板
│   └── NOTES.txt           # 安装后提示信息
└── .helmignore             # 打包时忽略的文件
```

## 四、模板语法详解

### 内置对象

| 对象 | 含义 | 示例 |
|------|------|------|
| `.Values` | values.yaml 中的值 | `{{ .Values.image.tag }}` |
| `.Release` | 当前 Release 信息 | `{{ .Release.Name }}` |
| `.Chart` | Chart.yaml 信息 | `{{ .Chart.AppVersion }}` |
| `.Capabilities` | 集群能力 | `{{ .Capabilities.KubeVersion }}` |

### 常用语法

```yaml
# 变量引用
image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"

# 默认值
port: {{ .Values.service.port | default 8080 }}

# 条件判断
{{- if .Values.ingress.enabled }}
apiVersion: networking.k8s.io/v1
kind: Ingress
# ...
{{- end }}

# 循环
{{- range .Values.env }}
- name: {{ .name }}
  value: {{ .value | quote }}
{{- end }}

# with（改变作用域）
{{- with .Values.resources }}
resources:
  {{- toYaml . | nindent 2 }}
{{- end }}

# include 辅助模板
labels:
  {{- include "game-chart.labels" . | nindent 4 }}
```

## 五、完整实操：封装游戏服务 Chart

### Chart.yaml

```yaml
apiVersion: v2
name: game-server
description: Game server Helm chart
type: application
version: 0.1.0        # Chart 版本
appVersion: "1.0.0"   # 应用版本
```

### values.yaml（默认值）

```yaml
# 全局配置
replicaCount: 2

image:
  repository: game-gate
  tag: "v1.0"
  pullPolicy: IfNotPresent

service:
  type: ClusterIP
  port: 80

resources:
  requests:
    cpu: "100m"
    memory: "128Mi"
  limits:
    cpu: "500m"
    memory: "256Mi"

autoscaling:
  enabled: false
  minReplicas: 2
  maxReplicas: 10
  targetCPUUtilization: 60

ingress:
  enabled: false
  host: ""
  tls: false

config:
  serverName: "game-gate"
  maxConnections: 10000
  redis:
    addr: "redis:6379"
  mysql:
    host: "mysql"
    port: 3306
    database: "gamedb"

secret:
  mysqlPassword: ""    # 通过 --set 或环境配置注入
```

### templates/_helpers.tpl

```yaml
{{/* Chart 全名 */}}
{{- define "game-chart.fullname" -}}
{{- printf "%s-%s" .Release.Name .Chart.Name | trunc 63 | trimSuffix "-" }}
{{- end }}

{{/* 通用标签 */}}
{{- define "game-chart.labels" -}}
app.kubernetes.io/name: {{ .Chart.Name }}
app.kubernetes.io/instance: {{ .Release.Name }}
app.kubernetes.io/version: {{ .Chart.AppVersion | quote }}
app.kubernetes.io/managed-by: {{ .Release.Service }}
{{- end }}

{{/* Selector 标签 */}}
{{- define "game-chart.selectorLabels" -}}
app.kubernetes.io/name: {{ .Chart.Name }}
app.kubernetes.io/instance: {{ .Release.Name }}
{{- end }}
```

### templates/configmap.yaml

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: {{ include "game-chart.fullname" . }}-config
  labels:
    {{- include "game-chart.labels" . | nindent 4 }}
data:
  server.yaml: |
    server:
      name: {{ .Values.config.serverName }}
      max_connections: {{ .Values.config.maxConnections }}
    redis:
      addr: {{ .Values.config.redis.addr }}
    mysql:
      host: {{ .Values.config.mysql.host }}
      port: {{ .Values.config.mysql.port }}
      database: {{ .Values.config.mysql.database }}
```

### templates/deployment.yaml

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ include "game-chart.fullname" . }}
  labels:
    {{- include "game-chart.labels" . | nindent 4 }}
spec:
  {{- if not .Values.autoscaling.enabled }}
  replicas: {{ .Values.replicaCount }}
  {{- end }}
  selector:
    matchLabels:
      {{- include "game-chart.selectorLabels" . | nindent 6 }}
  template:
    metadata:
      labels:
        {{- include "game-chart.selectorLabels" . | nindent 8 }}
    spec:
      containers:
      - name: {{ .Chart.Name }}
        image: "{{ .Values.image.repository }}:{{ .Values.image.tag }}"
        imagePullPolicy: {{ .Values.image.pullPolicy }}
        ports:
        - containerPort: 80
        resources:
          {{- toYaml .Values.resources | nindent 10 }}
        volumeMounts:
        - name: config
          mountPath: /app/config
          readOnly: true
      volumes:
      - name: config
        configMap:
          name: {{ include "game-chart.fullname" . }}-config
```

### templates/service.yaml

```yaml
apiVersion: v1
kind: Service
metadata:
  name: {{ include "game-chart.fullname" . }}
  labels:
    {{- include "game-chart.labels" . | nindent 4 }}
spec:
  type: {{ .Values.service.type }}
  selector:
    {{- include "game-chart.selectorLabels" . | nindent 4 }}
  ports:
  - port: {{ .Values.service.port }}
    targetPort: 80
    protocol: TCP
```

### templates/hpa.yaml

```yaml
{{- if .Values.autoscaling.enabled }}
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: {{ include "game-chart.fullname" . }}
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: {{ include "game-chart.fullname" . }}
  minReplicas: {{ .Values.autoscaling.minReplicas }}
  maxReplicas: {{ .Values.autoscaling.maxReplicas }}
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: {{ .Values.autoscaling.targetCPUUtilization }}
{{- end }}
```

## 六、Helm 命令操作

```bash
# 调试：渲染模板但不安装
helm template game ./game-chart -f values-dev.yaml

# 安装（开发环境）
helm install game-dev ./game-chart -f values-dev.yaml -n game-dev

# 安装（生产环境）
helm install game-prod ./game-chart -f values-prod.yaml -n game-prod

# 查看 Release
helm list -A
# NAME       NAMESPACE   REVISION  STATUS    CHART              APP VERSION
# game-dev   game-dev    1         deployed  game-server-0.1.0  1.0.0
# game-prod  game-prod   1         deployed  game-server-0.1.0  1.0.0

# 升级（修改副本数或镜像版本）
helm upgrade game-dev ./game-chart -f values-dev.yaml --set replicaCount=3

# 查看历史
helm history game-dev
# REVISION  STATUS      DESCRIPTION
# 1         superseded  Install complete
# 2         deployed    Upgrade complete

# 回滚
helm rollback game-dev 1

# 卸载
helm uninstall game-dev -n game-dev
```
