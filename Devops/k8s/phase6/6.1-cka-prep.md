| 项目 | 信息 |

## 四、kubectl 高效技巧（必须熟练）

### 基础配置

```bash
# 自动补全（考试环境已配置，但确认一下）
source <(kubectl completion bash)
alias k=kubectl
complete -o default -F __start_kubectl k

# 快速切换集群/上下文（考试常用）
kubectl config use-context <context-name>

# 设置默认命名空间
kubectl config set-context --current --namespace=<ns>
```

### dry-run 生成 YAML（核心技巧）

```bash
# 生成 Pod YAML
k run nginx --image=nginx:1.25 --dry-run=client -o yaml > pod.yaml

# 生成 Deployment YAML
k create deployment web --image=nginx --replicas=3 --dry-run=client -o yaml > deploy.yaml

# 生成 Service YAML
k expose deployment web --port=80 --target-port=80 --type=NodePort --dry-run=client -o yaml > svc.yaml

# 生成 Job YAML
k create job test-job --image=busybox -- /bin/sh -c "echo hello" --dry-run=client -o yaml > job.yaml

# 生成 CronJob YAML
k create cronjob test-cron --image=busybox --schedule="*/5 * * * *" -- /bin/sh -c "echo hello" --dry-run=client -o yaml > cron.yaml

# 生成 ConfigMap
k create configmap my-config --from-literal=key1=val1 --dry-run=client -o yaml > cm.yaml

# 生成 Secret
k create secret generic my-secret --from-literal=password=abc123 --dry-run=client -o yaml > secret.yaml

# 生成 ServiceAccount
k create sa my-sa --dry-run=client -o yaml > sa.yaml

# 生成 Role
k create role pod-reader --verb=get,list,watch --resource=pods --dry-run=client -o yaml > role.yaml

# 生成 RoleBinding
k create rolebinding pod-reader-binding --role=pod-reader --serviceaccount=default:my-sa --dry-run=client -o yaml > rb.yaml

# 生成 Ingress
k create ingress my-ingress --rule="host/path=svc:port" --dry-run=client -o yaml > ingress.yaml
```

### 快捷查询

```bash
# 快速查看资源（宽输出）
k get pods -o wide
k get all

# JSON Path 提取信息
k get nodes -o jsonpath='{.items[*].status.addresses[?(@.type=="InternalIP")].address}'

# 按条件排序
k get pods --sort-by='.metadata.creationTimestamp'
k get pv --sort-by='.spec.capacity.storage'

# 查看资源 API 字段
k explain pod.spec.containers.livenessProbe
k explain deployment.spec.strategy
```

## 五、高频考点速查模板

### 1. etcd 备份与恢复

```bash
# 备份
ETCDCTL_API=3 etcdctl snapshot save /tmp/etcd-backup.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key

# 恢复
ETCDCTL_API=3 etcdctl snapshot restore /tmp/etcd-backup.db \
  --data-dir=/var/lib/etcd-restored

# 修改 etcd 静态 Pod 的 data-dir 路径
vim /etc/kubernetes/manifests/etcd.yaml
# 修改 --data-dir=/var/lib/etcd-restored
# 修改 volumeMounts 和 hostPath
```

### 2. kubeadm 集群升级

```bash
# 升级控制面
apt-get update
apt-get install -y kubeadm=1.29.0-1.1
kubeadm upgrade plan
kubeadm upgrade apply v1.29.0

# 升级 kubelet 和 kubectl
apt-get install -y kubelet=1.29.0-1.1 kubectl=1.29.0-1.1
systemctl daemon-reload
systemctl restart kubelet

# 升级 Worker 节点
kubectl drain <node> --ignore-daemonsets --delete-emptydir-data
# 在 Worker 节点上执行 apt-get install + kubeadm upgrade node
kubectl uncordon <node>
```

### 3. 节点故障排查

```bash
# 检查节点状态
k get nodes
k describe node <node>

# 常见问题
# kubelet 未运行
ssh <node>
systemctl status kubelet
systemctl start kubelet
systemctl enable kubelet

# 证书问题
journalctl -u kubelet -f
```

### 4. Pod 故障排查

```bash
# 常用排查命令
k describe pod <pod>      # 查看事件
k logs <pod> [-c container]  # 查看日志
k logs <pod> --previous      # 查看上次崩溃的日志
k get events --sort-by='.lastTimestamp'
```

### 5. NetworkPolicy

```bash
# 快速生成：参考官方文档
# https://kubernetes.io/docs/concepts/services-networking/network-policies/
```

### 6. PV/PVC

```bash
# 查看绑定状态
k get pv,pvc

# 常见问题：PVC Pending
# 检查 StorageClass、accessModes、capacity 是否匹配
```

### 7. Ingress

```bash
k create ingress my-ingress \
  --rule="myhost.com/app=my-svc:8080" \
  --annotation nginx.ingress.kubernetes.io/rewrite-target=/ \
  --dry-run=client -o yaml > ingress.yaml
```

### 8. Sidecar 容器

```yaml
spec:
  containers:
  - name: main
    image: nginx
  - name: sidecar
    image: busybox
    command: ["sh", "-c", "tail -f /var/log/nginx/access.log"]
    volumeMounts:
    - name: shared
      mountPath: /var/log/nginx
  volumes:
  - name: shared
    emptyDir: {}
```

## 六、时间管理策略

| 策略 | 说明 |
|------|------|
| **先易后难** | 快速完成简单题，跳过难题最后回来 |
| **注意权重** | 每题有分值标注，优先做高分题 |
| **切换上下文** | 题目会指定集群，别忘了 `kubectl config use-context` |
| **用 dry-run** | 不要手写 YAML，用命令生成后修改 |
| **善用文档** | kubernetes.io 搜索框很好用 |
| **5 分钟规则** | 一道题卡住 5 分钟就跳过 |
