# 3.1 StatefulSet — 有状态游戏服务

> 本节目标：掌握 StatefulSet 的有序部署机制，实现有状态游戏服务编排。

---

## 一、StatefulSet vs Deployment

| 特性 | Deployment | StatefulSet |
|------|-----------|-------------|
| Pod 名称 | 随机后缀（`app-6d8f9-abc12`） | 有序编号（`app-0, app-1, app-2`） |
| 创建顺序 | 并行 | 有序（0→1→2） |
| 删除顺序 | 并行 | 逆序（2→1→0） |
| 网络标识 | 不稳定 | 稳定（通过 Headless Service） |
| 存储 | 共享 PVC | 每个 Pod 独立 PVC（volumeClaimTemplates） |
| 适用场景 | 无状态服务 | 数据库、有状态游戏服务、消息队列 |

---

## 二、核心特性

### 1. 有序部署/删除

- **创建**：Pod-0 Ready 后才创建 Pod-1，Pod-1 Ready 后才创建 Pod-2
- **删除/缩容**：Pod-2 终止后才终止 Pod-1
- **更新**：默认 RollingUpdate，逆序更新（从最后一个 Pod 开始）

### 2. 稳定网络标识

Pod DNS 格式：`<pod-name>.<headless-service>.<namespace>.svc.cluster.local`

```
game-room-0.game-room-svc.default.svc.cluster.local
game-room-1.game-room-svc.default.svc.cluster.local
game-room-2.game-room-svc.default.svc.cluster.local
```

### 3. Headless Service

`clusterIP: None` 的 Service，不分配 ClusterIP，DNS 直接解析到 Pod IP。

```
普通 Service:  DNS → ClusterIP → kube-proxy 负载均衡 → Pod
Headless Service: DNS → 直接返回所有 Pod IP（A 记录）
                  + 每个 Pod 有独立 DNS 记录
```

### 4. volumeClaimTemplates

为每个 Pod 自动创建独立的 PVC：

```
StatefulSet (replicas: 3)
├── game-room-0 → PVC: data-game-room-0
├── game-room-1 → PVC: data-game-room-1
└── game-room-2 → PVC: data-game-room-2
```

Pod 被删除重建后，仍然绑定到同一个 PVC。

## 三、完整实操：部署游戏房间服务

### YAML 文件

```yaml
# game-room-statefulset.yaml

# Headless Service
apiVersion: v1
kind: Service
metadata:
  name: game-room-svc
spec:
  clusterIP: None          # Headless
  selector:
    app: game-room
  ports:
  - port: 8080
    targetPort: 8080
    name: game

---
# StatefulSet
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: game-room
spec:
  serviceName: game-room-svc    # 必须指定 Headless Service
  replicas: 3
  selector:
    matchLabels:
      app: game-room
  template:
    metadata:
      labels:
        app: game-room
    spec:
      containers:
      - name: room
        image: nginx:1.25
        ports:
        - containerPort: 8080
          name: game
        volumeMounts:
        - name: room-data
          mountPath: /data
        # 写入标识文件用于验证
        command:
        - /bin/sh
        - -c
        - |
          echo "I am $(hostname)" > /usr/share/nginx/html/index.html
          nginx -g 'daemon off;'
  volumeClaimTemplates:
  - metadata:
      name: room-data
    spec:
      accessModes: ["ReadWriteOnce"]
      resources:
        requests:
          storage: 1Gi
```

### 部署与验证

```bash
# 部署
kubectl apply -f game-room-statefulset.yaml

# 观察有序创建过程
kubectl get pods -l app=game-room -w
# NAME          READY   STATUS    RESTARTS   AGE
# game-room-0   1/1     Running   0          10s
# game-room-1   1/1     Running   0          20s   ← 0 Ready 后才创建
# game-room-2   1/1     Running   0          30s   ← 1 Ready 后才创建

# 查看 PVC（每个 Pod 一个）
kubectl get pvc
# NAME                    STATUS   VOLUME         CAPACITY
# room-data-game-room-0   Bound    pvc-xxx        1Gi
# room-data-game-room-1   Bound    pvc-yyy        1Gi
# room-data-game-room-2   Bound    pvc-zzz        1Gi
```

### 验证 DNS 解析

```bash
# 启动一个临时 Pod 用于 DNS 测试
kubectl run dns-test --rm -it --image=busybox:1.36 --restart=Never -- sh

# 在临时 Pod 内执行：
# 解析单个 Pod
nslookup game-room-0.game-room-svc
# Server:    10.96.0.10
# Address 1: 10.96.0.10 kube-dns.kube-system.svc.cluster.local
# Name:      game-room-0.game-room-svc
# Address 1: 10.244.0.5 game-room-0.game-room-svc.default.svc.cluster.local  ✅

# 解析整个 Headless Service（返回所有 Pod IP）
nslookup game-room-svc
# Name:      game-room-svc
# Address 1: 10.244.0.5
# Address 2: 10.244.0.6
# Address 3: 10.244.0.7

# 通过 DNS 名访问特定 Pod
wget -qO- http://game-room-0.game-room-svc:8080
# I am game-room-0

wget -qO- http://game-room-1.game-room-svc:8080
# I am game-room-1
```

### 验证 Pod 名称稳定性

```bash
# 记录当前 Pod IP
kubectl get pods -l app=game-room -o wide
# game-room-0   10.244.0.5   node1
# game-room-1   10.244.0.6   node1
# game-room-2   10.244.0.7   node1

# 删除 Pod（模拟故障）
kubectl delete pod game-room-1

# 观察重建
kubectl get pods -l app=game-room -w
# game-room-1   1/1     Terminating   0          5m
# game-room-1   0/1     Pending       0          0s     ← 同名重建
# game-room-1   1/1     Running       0          5s     ✅ 名称不变

# 验证数据保留（PVC 仍绑定到 game-room-1）
kubectl get pvc room-data-game-room-1
# STATUS: Bound  ✅
```
