# 3.3 网络策略与 Ingress

> 本节目标：掌握 NetworkPolicy 网络隔离和 Ingress 路由配置。

---

## 一、NetworkPolicy

### 概念

NetworkPolicy 是 K8s 的**网络防火墙**，控制 Pod 间的流量。

**默认行为**：未配置 NetworkPolicy 时，所有 Pod 间可自由通信。一旦对某个 Pod 应用了 NetworkPolicy，**未明确允许的流量将被拒绝**。

> 注意：NetworkPolicy 需要 CNI 插件支持（如 Calico、Cilium），Flannel 不支持。

### YAML 结构

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: policy-name
spec:
  podSelector:          # 策略作用于哪些 Pod
    matchLabels:
      app: game
  policyTypes:
  - Ingress             # 入站规则
  - Egress              # 出站规则
  ingress:              # 允许哪些来源访问
  - from:
    - podSelector:      # 来源 Pod
        matchLabels:
          app: gate
    - namespaceSelector: # 来源命名空间
        matchLabels:
          env: prod
    - ipBlock:          # 来源 IP 段
        cidr: 10.0.0.0/8
    ports:
    - protocol: TCP
      port: 8080
  egress:               # 允许访问哪些目标
  - to:
    - podSelector:
        matchLabels:
          app: redis
    ports:
    - protocol: TCP
      port: 6379
```

### 注意：`from` 列表中的 `-` 含义

```yaml
# AND 关系（同一个 from 项）
ingress:
- from:
  - podSelector:
      matchLabels: {app: gate}
    namespaceSelector:
      matchLabels: {env: prod}
  # = prod 命名空间中 label 为 app=gate 的 Pod

# OR 关系（不同 from 项）
ingress:
- from:
  - podSelector:
      matchLabels: {app: gate}
  - namespaceSelector:
      matchLabels: {env: prod}
  # = app=gate 的 Pod，或者 prod 命名空间的所有 Pod
```

## 二、Ingress

### 概念

Ingress 是 K8s 的 **HTTP(S) 反向代理入口**，将外部流量路由到集群内的 Service。

```
互联网 → Ingress Controller (Nginx/Traefik) → Ingress 规则 → Service → Pod
```

### Ingress Controller

Ingress 资源本身只是规则声明，需要 **Ingress Controller** 来实现：

| Controller | 特点 |
|-----------|------|
| Nginx Ingress | 最常用，社区版和企业版 |
| Traefik | 云原生，自动发现 |
| Kong | API 网关功能丰富 |

### 安装 Nginx Ingress Controller

```bash
# minikube
minikube addons enable ingress

# kind（需要特殊配置）
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml

# 通用安装（Helm）
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm install ingress-nginx ingress-nginx/ingress-nginx

# 验证
kubectl get pods -n ingress-nginx
# NAME                                        READY   STATUS
# ingress-nginx-controller-xxx                1/1     Running
```

### pathType 类型

| 类型 | 匹配规则 | 示例 |
|------|----------|------|
| Exact | 精确匹配 | `/api/gate` 只匹配 `/api/gate` |
| Prefix | 前缀匹配 | `/api/gate` 匹配 `/api/gate`、`/api/gate/xxx` |
| ImplementationSpecific | 取决于 IngressClass | 由 Controller 决定 |

## 三、完整实操

### 实操 1：配置 Ingress 路由

先部署两个服务：

```yaml
# game-services.yaml

# Gate Service
apiVersion: apps/v1
kind: Deployment
metadata:
  name: gate
spec:
  replicas: 2
  selector:
    matchLabels:
      app: gate
  template:
    metadata:
      labels:
        app: gate
    spec:
      containers:
      - name: gate
        image: nginx:1.25
        ports:
        - containerPort: 80
        command: ["/bin/sh", "-c"]
        args:
        - |
          echo 'Gate Service' > /usr/share/nginx/html/index.html
          mkdir -p /usr/share/nginx/html/api/gate
          echo 'Gate API Response' > /usr/share/nginx/html/api/gate/index.html
          nginx -g 'daemon off;'
---
apiVersion: v1
kind: Service
metadata:
  name: gate-service
spec:
  selector:
    app: gate
  ports:
  - port: 80

---
# Login Service
apiVersion: apps/v1
kind: Deployment
metadata:
  name: login
spec:
  replicas: 2
  selector:
    matchLabels:
      app: login
  template:
    metadata:
      labels:
        app: login
    spec:
      containers:
      - name: login
        image: nginx:1.25
        ports:
        - containerPort: 80
        command: ["/bin/sh", "-c"]
        args:
        - |
          echo 'Login Service' > /usr/share/nginx/html/index.html
          mkdir -p /usr/share/nginx/html/api/login
          echo 'Login API Response' > /usr/share/nginx/html/api/login/index.html
          nginx -g 'daemon off;'
---
apiVersion: v1
kind: Service
metadata:
  name: login-service
spec:
  selector:
    app: login
  ports:
  - port: 80
```

配置 Ingress 规则：

```yaml
# game-ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: game-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
  - host: game.local           # 可选，不配置则匹配所有
    http:
      paths:
      - path: /api/gate
        pathType: Prefix
        backend:
          service:
            name: gate-service
            port:
              number: 80
      - path: /api/login
        pathType: Prefix
        backend:
          service:
            name: login-service
            port:
              number: 80
```

```bash
kubectl apply -f game-services.yaml
kubectl apply -f game-ingress.yaml

# 获取 Ingress 地址
kubectl get ingress
# NAME           CLASS   HOSTS        ADDRESS        PORTS   AGE
# game-ingress   nginx   game.local   192.168.49.2   80      5s

# 测试（minikube）
# 添加 hosts 解析：echo "192.168.49.2 game.local" >> /etc/hosts
curl http://game.local/api/gate
# Gate API Response  ✅

curl http://game.local/api/login
# Login API Response  ✅
```

### TLS 配置

```bash
# 生成自签名证书
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout tls.key -out tls.crt -subj "/CN=game.local"

# 创建 TLS Secret
kubectl create secret tls game-tls --cert=tls.crt --key=tls.key
```

```yaml

  - hosts:
    - game.local
```

### 实操 2：NetworkPolicy 限制访问

```yaml
# game-network-policy.yaml

# 部署 game 服务
apiVersion: apps/v1
kind: Deployment
metadata:
  name: game
spec:
  replicas: 2
  selector:
    matchLabels:
      app: game
  template:
    metadata:
      labels:
        app: game
    spec:
      containers:
      - name: game
        image: nginx:1.25
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: game-service
spec:
  selector:
    app: game
  ports:
  - port: 80

---
# NetworkPolicy：只允许 gate Pod 访问 game Pod
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: game-allow-gate-only
spec:
  podSelector:
    matchLabels:
      app: game            # 保护 game Pod
  policyTypes:
  - Ingress
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: gate        # 只允许 gate 访问
    ports:
    - protocol: TCP
      port: 80
```

```bash
kubectl apply -f game-network-policy.yaml

# 验证：从 gate Pod 访问 game — 应成功 ✅
kubectl exec deploy/gate -- curl -s --max-time 3 http://game-service
# ... nginx 默认页面 ...

# 验证：从 login Pod 访问 game — 应被拒绝 ❌
kubectl exec deploy/login -- curl -s --max-time 3 http://game-service
# curl: (28) Connection timed out  ✅（被 NetworkPolicy 阻止）

# 创建一个无标签的测试 Pod
kubectl run test-pod --rm -it --image=busybox:1.36 -- wget -qO- --timeout=3 http://game-service
# wget: download timed out  ✅（被阻止）
```
