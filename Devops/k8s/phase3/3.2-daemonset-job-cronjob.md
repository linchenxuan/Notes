# 3.2 DaemonSet / Job / CronJob

> 本节目标：掌握三种特殊工作负载的使用场景和配置。

---

## 一、DaemonSet

### 概念

DaemonSet 确保**每个（或指定）Node 上运行且仅运行一个 Pod**。

### 使用场景

- 日志采集（Fluent Bit、Filebeat）
- 监控 agent（Node Exporter、Datadog Agent）
- 网络插件（Calico、Flannel 的 Node 组件）
- 存储插件（CSI Node Driver）

### DaemonSet vs Deployment

| 特性 | DaemonSet | Deployment |
|------|-----------|------------|
| Pod 分布 | 每个 Node 一个 | Scheduler 自由调度 |
| 副本数 | 由 Node 数决定 | 手动指定 replicas |
| 新增 Node | 自动部署 Pod | 不自动 |
| 适用场景 | Node 级别守护进程 | 无状态应用 |

### 实操：DaemonSet 部署 Fluent Bit

```yaml
# fluent-bit-daemonset.yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluent-bit
  namespace: kube-system
  labels:
    app: fluent-bit
spec:
  selector:
    matchLabels:
      app: fluent-bit
  template:
    metadata:
      labels:
        app: fluent-bit
    spec:
      containers:
      - name: fluent-bit
        image: fluent/fluent-bit:2.2
        resources:
          requests:
            cpu: "50m"
            memory: "64Mi"
          limits:
            cpu: "200m"
            memory: "128Mi"
        volumeMounts:
        - name: varlog
          mountPath: /var/log
          readOnly: true
        - name: containers-log
          mountPath: /var/lib/docker/containers
          readOnly: true
      volumes:
      - name: varlog
        hostPath:
          path: /var/log
      - name: containers-log
        hostPath:
          path: /var/lib/docker/containers
      tolerations:              # 允许调度到 master 节点
      - key: node-role.kubernetes.io/control-plane
        effect: NoSchedule
```

```bash
kubectl apply -f fluent-bit-daemonset.yaml

# 验证每个 Node 有且仅有一个 Pod
kubectl get pods -n kube-system -l app=fluent-bit -o wide
# NAME              READY   STATUS    NODE
# fluent-bit-abc12  1/1     Running   node-1
# fluent-bit-def34  1/1     Running   node-2
# fluent-bit-ghi56  1/1     Running   control-plane

# 查看 DaemonSet 状态
kubectl get ds -n kube-system fluent-bit
# NAME        DESIRED   CURRENT   READY   UP-TO-DATE   AVAILABLE
# fluent-bit  3         3         3       3            3
```

## 二、Job

### 概念

Job 运行**一次性任务**，确保 Pod 成功完成指定次数后退出。

### 关键参数

| 参数 | 含义 | 默认值 |
|------|------|--------|
| completions | 需要成功完成的 Pod 数 | 1 |
| parallelism | 并行运行的 Pod 数 | 1 |
| backoffLimit | 失败重试次数 | 6 |
| activeDeadlineSeconds | 超时时间（秒） | 无 |
| ttlSecondsAfterFinished | 完成后自动清理时间 | 不清理 |

### 示例

```yaml
# data-migration-job.yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: data-migration
spec:
  completions: 1
  backoffLimit: 3
  ttlSecondsAfterFinished: 300    # 完成 5 分钟后自动删除
  template:
    spec:
      containers:
      - name: migrate
        image: alpine:3.19
        command:
        - /bin/sh
        - -c
        - |
          echo "Starting data migration..."
          echo "Processing player data..."
          sleep 5
          echo "Migration completed successfully!"
      restartPolicy: Never    # Job 必须是 Never 或 OnFailure
```

```bash
kubectl apply -f data-migration-job.yaml

# 查看 Job 状态
kubectl get jobs
# NAME              COMPLETIONS   DURATION   AGE
# data-migration    1/1           8s         30s

# 查看日志
kubectl logs job/data-migration
# Starting data migration...
# Processing player data...
# Migration completed successfully!
```

## 三、CronJob

### 概念

CronJob 按照**定时计划**创建 Job，类似 Linux crontab。

### Cron 表达式

```
┌───────── 分钟 (0-59)
│ ┌─────── 小时 (0-23)
│ │ ┌───── 日 (1-31)
│ │ │ ┌─── 月 (1-12)
│ │ │ │ ┌─ 星期几 (0-6, 0=周日)
│ │ │ │ │
* * * * *
```

常用示例：

| 表达式 | 含义 |
|--------|------|
| `0 2 * * *` | 每天凌晨 2:00 |
| `*/5 * * * *` | 每 5 分钟 |
| `0 0 * * 0` | 每周日 0:00 |
| `0 3 1 * *` | 每月 1 号 3:00 |
| `30 4 * * 1-5` | 工作日 4:30 |

### 关键参数

| 参数 | 含义 |
|------|------|
| concurrencyPolicy | Allow（允许并发）/ Forbid（跳过新的）/ Replace（取消旧的） |
| startingDeadlineSeconds | 错过调度后的容忍时间 |
| successfulJobsHistoryLimit | 保留成功 Job 数（默认 3） |
| failedJobsHistoryLimit | 保留失败 Job 数（默认 1） |

### 实操：每日游戏数据统计

```yaml
# game-stats-cronjob.yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: game-daily-stats
spec:
  schedule: "0 2 * * *"           # 每天凌晨 2:00
  concurrencyPolicy: Forbid        # 禁止并发
  successfulJobsHistoryLimit: 7    # 保留 7 天记录
  failedJobsHistoryLimit: 3
  jobTemplate:
    spec:
      backoffLimit: 2
      template:
        spec:
          containers:
          - name: stats
            image: alpine:3.19
            command:
            - /bin/sh
            - -c
            - |
              echo "=== Game Daily Stats ==="
              echo "Date: $(date '+%Y-%m-%d')"
              echo ""
              echo "--- Player Statistics ---"
              echo "DAU: $((RANDOM % 10000 + 5000))"
              echo "New Players: $((RANDOM % 1000 + 200))"
              echo "Peak Online: $((RANDOM % 5000 + 3000))"
              echo ""
              echo "--- Match Statistics ---"
              echo "Total Matches: $((RANDOM % 50000 + 20000))"
              echo "Avg Duration: $((RANDOM % 600 + 300))s"
              echo "Avg Wait Time: $((RANDOM % 30 + 5))s"
              echo ""
              echo "Stats collection completed!"
          restartPolicy: OnFailure
```

```bash
kubectl apply -f game-stats-cronjob.yaml

# 查看 CronJob
kubectl get cronjob
# NAME              SCHEDULE    SUSPEND   ACTIVE   LAST SCHEDULE
# game-daily-stats  0 2 * * *   False     0        <none>

# 手动触发一次（不等到凌晨 2 点）
kubectl create job --from=cronjob/game-daily-stats manual-stats-run

# 查看 Job 执行
kubectl get jobs
# NAME                          COMPLETIONS   DURATION   AGE
# manual-stats-run              1/1           3s         10s

# 查看输出
kubectl logs job/manual-stats-run
# === Game Daily Stats ===
# Date: 2024-01-15
# --- Player Statistics ---
# DAU: 8234
# New Players: 567
# Peak Online: 4521
# --- Match Statistics ---
# Total Matches: 35678
# Avg Duration: 458s
# Avg Wait Time: 12s
# Stats collection completed!

# 查看历史 Job 记录
kubectl get jobs --sort-by=.metadata.creationTimestamp
```
